TOPIC 13

YOLO는 왜 빨라?

Q. 실시간 탐지는 어떻게 가능해?

Part 1. YOLO가 뭐야?

YOLO가 뭔데 자꾸 얘기해?

You Only Look Once의 약자야.

"한 번만 봐도 된다"는 뜻이지.

YOLO의 핵심 아이디어: 기존 방식 (R-CNN): → 후보 영역 2000개 생성 → 각 영역마다 CNN 실행 → 2000번 계산 필요 → 느림! YOLO: → 이미지 전체를 1번만 CNN에 통과 → 한 번에 모든 박스 + 클래스 예측 → 계산 1번 → 빠름!

이름처럼 "한 번만 본다"는 게 핵심이야.

Part 2. 어떻게 작동해?

구체적으로 어떻게 한 번에 처리해?

그리드 시스템을 써.

YOLO 작동 원리: Step 1: 이미지를 그리드로 나누기 → 예: 7×7 = 49개 셀 Step 2: 각 셀이 예측 → "내 영역에 물체 중심이 있나?" → 있으면 박스 좌표 + 클래스 예측 Step 3: 각 셀마다 여러 개 앵커 박스 → 셀 하나당 2~3개 박스 예측 → 다양한 크기/비율 대응 Step 4: NMS로 중복 제거 → 신뢰도 높은 것만 남김 출력: → 49개 셀 × 2개 박스 = 98개 예측 → NMS 후 최종 5~10개 박스

이해됐어?

이미지를 격자로 나눠서, 각 칸이 동시에 예측하는 거야.

Part 3. 왜 빠를까?

정확히 뭐가 빠르게 만들어?

여러 최적화가 결합됐어.

방식 R-CNN YOLO
CNN 실행 횟수 2000번 1번
처리 시간 ~1초/이미지 ~0.02초/이미지 (50 FPS)
파이프라인 여러 단계 단일 네트워크
병렬화 어려움 쉬움 (GPU 최적화)
속도 비교: Faster R-CNN: → 7 FPS (초당 7프레임) → 높은 정확도 → 자율주행 X YOLO v3: → 30~60 FPS → 괜찮은 정확도 → 실시간 가능! YOLO v5/v7/v8: → 100+ FPS (경량 버전) → 정확도도 향상 → 엣지 디바이스에서도 작동

Part 4. YOLO의 발전

YOLO 버전 역사: YOLOv1 (2015): → 최초 버전 → 속도 혁신, 정확도 낮음 YOLOv2 (2016): → Batch Normalization 추가 → 앵커 박스 도입 → 정확도 개선 YOLOv3 (2018): → FPN (Feature Pyramid Network) → 다양한 크기 객체 잘 탐지 → 실무 표준 YOLOv4 (2020): → CSPNet, Mish 활성화 함수 → 정확도 대폭 향상 YOLOv5 (2020): → PyTorch 구현 → 사용 편의성 ↑ → 가장 인기 YOLOv7/v8 (2022-2023): → 최신 기술 집약 → 정확도 + 속도 모두 개선 → Transformer 요소 도입

YOLO는 계속 진화 중이야.

속도는 유지하면서 정확도는 계속 올라가고 있어.

Part 5. YOLO의 한계

YOLO도 단점이 있어?

당연히 있지.

YOLO의 약점: 1. 작은 객체 탐지 → 그리드 크기 때문에 작은 물체 놓침 → 예: 멀리 있는 사람 2. 겹친 객체 → 한 셀에 물체 중심이 2개면? → 하나만 탐지할 수 있음 3. 새로운 종횡비 → 학습 때 안 본 비율의 물체 → 정확도 떨어짐 해결책: → YOLOv3 이후: 여러 스케일 탐지 → 더 작은 그리드 사용 (13×13, 26×26, 52×52) → 많이 개선됐지만 완벽하진 않음

Part 6. 실전 활용

YOLO 활용 분야: 자율주행: → 실시간 차량, 보행자 탐지 → 30 FPS 이상 필수 → YOLO 최적 스마트폰 앱: → 제한된 하드웨어 → 경량 YOLO (Tiny-YOLO) → 실시간 AR 가능 드론: → 비행 중 객체 추적 → 빠른 처리 필수 스포츠 분석: → 선수, 공 실시간 추적 → 60 FPS 영상 처리 CCTV: → 여러 대 동시 처리 → 효율성 중요 코드 예시: from ultralytics import YOLO model = YOLO('yolov8n.pt') # nano 버전 results = model('image.jpg') # 한 줄로 탐지 완료!

실시간이 필요하면 YOLO야.

속도가 생명인 곳에서는 선택지가 아니라 필수지.

핵심 정리

  • You Only Look Once - 한 번만 봐서 모든 객체 탐지
  • 그리드 시스템 - 이미지를 격자로 나눠 동시 예측
  • 압도적 속도 - 50~100 FPS, 실시간 가능
  • 단일 네트워크 - CNN 1번 실행으로 모든 예측
  • 계속 진화 - v1~v8, 정확도도 크게 향상
  • 실시간 표준 - 자율주행, 드론 등 필수 기술
