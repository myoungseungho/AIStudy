TOPIC 01

컴퓨터는 이미지를 어떻게 봐?

Q. 사진이 숫자 덩어리라고?

Part 1. 사람이 보는 것 vs 컴퓨터가 보는 것

컴퓨터는 이미지를 어떻게 인식하는 거야?

먼저 사람이 사진을 보는 방식부터 생각해보자.

고양이 사진을 보면 우리는 "귀여운 고양이다"라고 바로 인식하지?

하지만 컴퓨터는 전혀 그렇게 보지 않아.

그럼 컴퓨터는 뭘 보는데?

숫자 덩어리.

그게 전부야.

사람이 보는 것: 🐱 귀여운 고양이 → 털, 수염, 귀, 눈을 종합적으로 인식 컴퓨터가 보는 것: [ [245, 243, 240, 238, 235, ...], [240, 238, 235, 233, 230, ...], [235, 233, 230, 228, 225, ...], ... ] → 그냥 숫자 배열

이해됐어?

컴퓨터에게 이미지는 숫자로 된 표야.

"고양이"라는 개념은 없어. 오직 숫자의 패턴만 있을 뿐이지.

Part 2. 이미지는 작은 점들의 집합

근데 왜 숫자로 표현되는 거야?

이미지를 아주 확대해서 보면 작은 점들로 이루어져 있어.

이 점 하나하나를 "픽셀(pixel)"이라고 해.

비유: 모자이크 벽화를 생각해봐. 멀리서 보면: → 하나의 그림 가까이 보면: → 작은 타일 조각들의 집합 이미지도 똑같아. 픽셀 = 디지털 이미지의 작은 타일 조각
그래서 픽셀 하나가 숫자 하나인 거야?

정확해!

각 픽셀은 밝기 값이나 색상 값을 숫자로 가지고 있어.

흑백 이미지의 경우: 픽셀값 0 = 완전 검은색 픽셀값 128 = 중간 회색 픽셀값 255 = 완전 하얀색 예시 (3x3 흑백 이미지): 0 128 255 128 255 128 255 128 0 → 컴퓨터는 이걸 9개의 숫자로 저장

Part 3. 해상도가 뭐길래?

1920x1080 같은 건 뭘 의미하는 거야?

가로 픽셀 개수 × 세로 픽셀 개수

1920x1080 = 가로 1920개, 세로 1080개의 점으로 이루어진 이미지

그럼 숫자가 몇 개야?

계산해보자.

1920 × 1080 = 2,073,600개

흑백 이미지라면 2백만 개가 넘는 숫자로 표현되는 거야.

해상도 픽셀 수 용도
28×28 784개 손글씨 인식 (MNIST)
224×224 50,176개 이미지 분류 학습용
1920×1080 (FHD) 2,073,600개 일반 영상
3840×2160 (4K) 8,294,400개 고화질 영상

해상도가 높을수록:

1. 더 많은 숫자가 필요함 (용량 증가)

2. 더 섬세한 표현 가능 (품질 향상)

3. AI가 처리하는 데 더 많은 계산 필요

Part 4. 숫자만 보고 고양이를 인식한다고?

그럼 컴퓨터는 숫자만 보고 어떻게 "이게 고양이야"라고 알아?

패턴을 학습해.

수천, 수만 장의 고양이 사진을 보여주면서 "이런 숫자 패턴이 고양이야"라고 배우는 거지.

학습 과정: Step 1: 고양이 사진 10,000장을 숫자로 변환 → 각각 수백만 개의 숫자 배열 Step 2: AI에게 반복 학습 "이 숫자 패턴 = 고양이" "저 숫자 패턴 = 강아지" Step 3: 새로운 이미지를 숫자로 변환 → 학습한 패턴과 비교 → "이 패턴은 고양이와 95% 유사해!"
그럼 AI는 "고양이"라는 개념을 이해하는 게 아니구나?

맞아. 전혀 이해하지 못해.

AI는 그냥 숫자 패턴의 통계적 유사성만 계산할 뿐이야.

"고양이가 귀엽다", "고양이는 야옹 소리를 낸다" 같은 건 전혀 몰라.

오직 "이 숫자 배열은 '고양이'라고 레이블된 데이터와 비슷하다"만 알 뿐.

Part 5. 이게 왜 중요한 거야?

그래서 이미지가 숫자라는 게 왜 중요해?

왜냐하면 숫자는 계산할 수 있거든.

숫자로 표현되면 할 수 있는 것: 1. 더하기, 빼기, 곱하기 가능 → 이미지 필터링, 밝기 조정 2. 통계 계산 가능 → 평균, 분산으로 특징 추출 3. 패턴 인식 가능 → AI 학습 4. 변환 가능 → 회전, 축소, 확대 5. 비교 가능 → 두 이미지가 얼마나 비슷한지 계산

이해됐어?

컴퓨터는 숫자만 다룰 수 있어.

그래서 모든 걸 숫자로 바꿔야 하는 거야.

이미지든, 소리든, 글자든 - 전부 숫자로 변환돼야 컴퓨터가 처리할 수 있지.

Part 6. 실제 예시를 보자

실제로는 어떻게 생겼는지 보여줘

간단한 예시를 보자.

아주 작은 5x5 흑백 이미지로 "숫자 1"을 표현한다면:

시각적 표현: ⬜⬛⬜⬜⬜ ⬛⬛⬜⬜⬜ ⬜⬛⬜⬜⬜ ⬜⬛⬜⬜⬜ ⬛⬛⬛⬜⬜ 컴퓨터가 보는 것: [ [255, 0, 255, 255, 255], [ 0, 0, 255, 255, 255], [255, 0, 255, 255, 255], [255, 0, 255, 255, 255], [ 0, 0, 0, 255, 255] ] → 25개의 숫자로 "1"을 표현

실제 AI 학습용 손글씨 데이터셋(MNIST)은 28×28 픽셀이야.

그럼 숫자 하나가 784개의 숫자로 표현되는 거지.

그리고 AI는 이 784개 숫자의 패턴을 학습해서 "아, 이건 1이구나"라고 인식하게 돼.

핵심 정리

  • 컴퓨터는 이미지를 숫자 배열로 본다 - 사람처럼 "보는" 게 아니라 숫자로 처리
  • 픽셀 = 이미지의 최소 단위 - 각 픽셀은 밝기나 색상을 숫자로 저장
  • 해상도 = 픽셀 개수 - 1920×1080은 가로 1920개, 세로 1080개의 점
  • 숫자로 바꾸면 계산 가능 - 덧셈, 뺄셈, 통계로 이미지 처리 가능
  • AI는 패턴을 학습한다 - "이런 숫자 배열 = 고양이"라는 패턴 인식
