TOPIC 12

객체 탐지는 분류랑 뭐가 달라?

Q. 사진 속에서 물체 위치까지 찾는다고?

Part 1. 분류 vs 탐지

객체 탐지(Object Detection)가 뭐야? 이미지 분류랑 뭐가 달라?

분류는 "무엇", 탐지는 "무엇 + 어디"야.

작업 이미지 분류 객체 탐지
질문 이 사진은 뭐야? 어디에 뭐가 있어?
출력 레이블 1개
"고양이"		 박스 + 레이블 여러 개
"고양이 (100,50,200,150)"
"개 (300,200,400,350)"
객체 수 1개 (전체 이미지) 여러 개
위치 정보 없음 있음 (Bounding Box)
예시: 사진: 공원에서 강아지 2마리, 고양이 1마리 이미지 분류: → "동물" 객체 탐지: → 강아지 (50, 100, 200, 300) → 강아지 (400, 150, 550, 320) → 고양이 (250, 200, 350, 280)

탐지가 훨씬 어려워.

"뭔지" 알아내는 것도 어려운데, "어디에 있는지"까지 찾아야 하니까.

Part 2. 바운딩 박스(Bounding Box)

바운딩 박스가 뭐야?

객체를 감싸는 사각형이야.

바운딩 박스 표현: 방법 1: (x1, y1, x2, y2) → 왼쪽 위 모서리 + 오른쪽 아래 모서리 → (100, 50, 200, 150) 방법 2: (x, y, w, h) → 왼쪽 위 모서리 + 너비 + 높이 → (100, 50, 100, 100) 방법 3: (cx, cy, w, h) → 중심점 + 너비 + 높이 → (150, 100, 100, 100) + 클래스 레이블 + 신뢰도 → "고양이, 0.95" (95% 확신)

AI는 이 박스를 예측해야 해.

"여기쯤에 고양이가 있을 것 같은데... 95% 확신!"

Part 3. 어떻게 탐지해?

AI는 어떻게 위치를 찾아?

크게 두 가지 방식이 있어.

방식 1: Two-Stage (R-CNN 계열) Step 1: Region Proposal → "여기 뭔가 있을 것 같은데?" 후보 영역 2000개 생성 Step 2: Classification + Box Refinement → 각 영역을 CNN으로 분류 → 박스 위치 미세 조정 대표 모델: → R-CNN (2014) → Fast R-CNN (2015) → Faster R-CNN (2015) 장점: 정확도 높음 단점: 느림 (영역마다 CNN 돌려야 함) 방식 2: One-Stage (YOLO, SSD) Step 1: 전체 이미지를 한 번에 처리 → CNN으로 특징 추출 → 동시에 박스 + 클래스 예측 대표 모델: → YOLO (You Only Look Once) → SSD (Single Shot Detector) 장점: 빠름 (실시간 가능) 단점: Two-Stage보다 정확도 낮음 (요즘은 거의 비슷)

Part 4. 어떻게 학습해?

박스 위치를 어떻게 학습시켜?

손실 함수가 두 가지야.

객체 탐지 손실: 1. Classification Loss (분류 손실) → "고양이다" vs "강아지다" → CrossEntropy Loss → 일반 분류와 동일 2. Localization Loss (위치 손실) → 예측 박스 vs 정답 박스 → 얼마나 떨어져 있는지 → L1 Loss 또는 Smooth L1 Loss Total Loss = Classification Loss + Localization Loss 학습 과정: → 정답: "고양이 (100, 50, 200, 150)" → 예측: "고양이 (110, 55, 195, 145)" (95% 확신) 분류 손실: 낮음 (고양이 맞춤) 위치 손실: 약간 있음 (10픽셀 차이) → 위치를 조금씩 조정하며 학습

Part 5. IoU (교집합/합집합 비율)

박스가 얼마나 정확한지는 어떻게 측정해?

IoU (Intersection over Union)를 써.

IoU 계산: 정답 박스: A 예측 박스: B IoU = (A와 B의 교집합 영역) / (A와 B의 합집합 영역) 예시: 정답: [0, 0, 100, 100] (넓이 10,000) 예측: [50, 50, 150, 150] (넓이 10,000) 교집합: [50, 50, 100, 100] (넓이 2,500) 합집합: 10,000 + 10,000 - 2,500 = 17,500 IoU = 2,500 / 17,500 = 0.14 (14%) 평가 기준: → IoU > 0.5: 보통 "맞춤"으로 인정 → IoU > 0.75: 높은 정확도 → IoU > 0.9: 거의 완벽

Part 6. NMS (Non-Maximum Suppression)

한 객체에 박스가 여러 개 그려지면 어떡해?

NMS로 중복 제거해.

문제: 고양이 하나에 박스 3개 예측 → Box A: 신뢰도 0.95 → Box B: 신뢰도 0.87 → Box C: 신뢰도 0.92 → 거의 비슷한 위치 NMS 알고리즘: 1. 신뢰도 순으로 정렬 → A (0.95), C (0.92), B (0.87) 2. 가장 높은 것 선택 → A 선택 3. A와 IoU가 높은 박스 제거 → B, C는 A와 IoU > 0.5 → B, C 제거 4. 다음으로 높은 것 처리 → 반복 결과: 박스 하나만 남음!

Part 7. 실전 활용

객체 탐지 활용 사례: 자율주행: → 차, 사람, 신호등 탐지 → 실시간 처리 필수 (YOLO) 보안 CCTV: → 침입자, 수상한 행동 탐지 → 24시간 작동 공장 자동화: → 불량품 탐지 → 높은 정확도 필요 (Faster R-CNN) 스포츠 분석: → 선수, 공 추적 → 실시간 + 정확도 의료: → X-ray에서 병변 찾기 → 정밀도 최우선

분류보다 훨씬 실용적이야.

"뭐가 있는지"만 아는 것보다 "어디에 있는지"까지 알아야 행동할 수 있으니까.

핵심 정리

  • 분류 vs 탐지 - "무엇" vs "무엇 + 어디"
  • 바운딩 박스 - 객체를 감싸는 사각형 좌표
  • Two-Stage vs One-Stage - 정확도 vs 속도
  • 두 가지 손실 - 분류 손실 + 위치 손실
  • IoU - 박스 정확도 측정 지표
  • NMS - 중복 박스 제거
