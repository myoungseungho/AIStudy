TOPIC 15

GAN은 어떻게 가짜를 만들어?

Q. 생성자와 판별자가 싸운다는 게 뭐야?

Part 1. GAN이 뭐야?

GAN이 뭐야?

Generative Adversarial Network (생성적 적대 신경망)

두 개의 신경망이 경쟁하면서 학습하는 거야.

비유: 위조지폐범 vs 경찰 생성자 (Generator): → 위조지폐를 만드는 사람 → "진짜같은 가짜" 만들기 시도 판별자 (Discriminator): → 위조지폐를 잡는 경찰 → "진짜 vs 가짜" 구분 시도 서로 경쟁: → 위조범: 더 정교한 가짜 제작 → 경찰: 더 예리한 감별 능력 개발 → 둘 다 점점 실력 향상 최종 결과: → 위조범이 너무 잘 만들어서 → 경찰도 구분 못 하는 수준 → = 진짜 같은 가짜 완성!

적대적 경쟁으로 서로 발전하는 게 핵심이야.

Part 2. 어떻게 작동해?

구체적으로 어떻게 싸워?

단계별로 보자.

GAN 학습 과정: Step 1: 생성자가 가짜 이미지 생성 → 입력: 랜덤 노이즈 (100차원 벡터) → 출력: 가짜 이미지 (64×64×3) Step 2: 판별자에게 진짜 + 가짜 섞어서 제시 → 진짜 이미지 (실제 고양이 사진) → 가짜 이미지 (생성자가 만든 것) Step 3: 판별자가 구분 시도 → 진짜: "Real, 확률 0.9" → 가짜: "Fake, 확률 0.1" Step 4: 판별자 학습 → "진짜를 진짜로, 가짜를 가짜로 잘 맞춰야 해" → 손실 함수로 가중치 업데이트 Step 5: 생성자 학습 → "판별자를 속여야 해!" → 판별자가 "Real"이라고 착각하도록 학습 → 가짜를 더 진짜 같게 만듦 반복 → 수천 번 → 진짜 같은 가짜 완성

이해됐어?

생성자는 속이고, 판별자는 잡아내고 - 이 싸움이 핵심이야.

Part 3. 손실 함수

둘의 목표가 완전히 반대인데, 어떻게 학습해?

Min-Max 게임이야.

판별자의 목표: → 진짜를 1로, 가짜를 0으로 분류 → Loss_D = -log(D(real)) - log(1-D(fake)) → 이 손실을 최소화 생성자의 목표: → 가짜를 1로 분류되게 (판별자 속이기) → Loss_G = -log(D(fake)) → 판별자가 헷갈리게 만들기 수학적으로: 판별자: Loss 최소화 (Minimize) 생성자: 판별자 Loss 최대화 (Maximize) → Min-Max Game 균형점 (Nash Equilibrium): → 판별자: "50% 확률로 진짜/가짜" (구분 불가) → 생성자: 완벽한 가짜 생성 → 더 이상 발전 없음 = 학습 완료

Part 4. 왜 어려워?

GAN이 학습하기 어렵다고 들었는데, 왜?

균형 맞추기가 너무 어려워.

GAN의 문제점: 1. Mode Collapse (모드 붕괴) → 생성자가 한 가지만 만듦 → 예: 고양이 1종류만 계속 생성 → 다양성 상실 2. 학습 불안정 → 판별자가 너무 강하면? → 생성자가 학습 못 함 (그래디언트 소실) → 생성자가 너무 강하면? → 판별자가 무의미해짐 3. 하이퍼파라미터 민감 → Learning rate 조금만 바꿔도 실패 → 수백 번 실험 필요 4. 평가 어려움 → "좋은 이미지"의 기준이 모호 → 정량적 평가 지표 부족 → 그래서 학습이 예술!

GAN 학습은 "줄타기"와 같아.

두 모델의 균형을 계속 맞춰야 해.

Part 5. GAN의 발전

주요 GAN 변형: DCGAN (2015): → 합성곱 사용 → 안정적 학습 → GAN의 표준 StyleGAN (2018-2019): → NVIDIA 개발 → "이 사람은 존재하지 않습니다" → 초고해상도 얼굴 생성 CycleGAN (2017): → 이미지 변환 (말↔얼룩말) → 짝 데이터 불필요 Pix2Pix (2016): → 스케치 → 사실적 이미지 → 흑백 → 컬러 BigGAN (2018): → 대규모 학습 → ImageNet 수준 이미지 생성 Conditional GAN (cGAN): → 원하는 클래스 지정 가능 → "고양이 생성해줘" 같은 제어

Part 6. 실전 활용

GAN 활용 분야: 이미지 생성: → 존재하지 않는 얼굴 → ThisPersonDoesNotExist.com → 게임 캐릭터 자동 생성 이미지 변환: → 낮 → 밤 → 여름 → 겨울 → 스케치 → 사실적 그림 초해상도 (Super Resolution): → 저화질 → 고화질 → 오래된 사진 복원 데이터 증강: → 학습 데이터 부족 시 → 가짜 데이터 생성해서 추가 아트: → Artbreeder → AI가 생성한 예술작품 → NFT 아트 딥페이크: → 얼굴 바꾸기 → 목소리 합성 → (윤리적 문제 있음) 최신 트렌드: → Diffusion Model이 GAN 대체 중 → Stable Diffusion, DALL-E 3 → 더 안정적이고 품질 좋음

GAN은 "창조"를 가능하게 했어.

이제 AI가 예술가가 될 수 있는 시대야.

Part 7. GAN의 미래

GAN의 미래는?

사실 요즘은 Diffusion Model이 대세야.

GAN vs Diffusion: GAN의 장점: → 빠른 생성 속도 → Sharp한 이미지 GAN의 단점: → 학습 불안정 → Mode Collapse → 제어 어려움 Diffusion의 장점: → 안정적 학습 → 높은 품질 → 세밀한 제어 가능 Diffusion의 단점: → 생성 느림 (수십 스텝) 결론: → 품질 중요: Diffusion (Stable Diffusion, DALL-E) → 속도 중요: GAN (실시간 변환) → 둘 다 계속 발전 중

GAN은 생성 AI의 선구자야.

지금의 Stable Diffusion, Midjourney도 GAN 없었으면 불가능했을 거야.

핵심 정리

  • 생성자 vs 판별자 - 두 신경망의 적대적 경쟁
  • Min-Max 게임 - 서로 반대 목표로 학습
  • 균형이 핵심 - 한쪽이 너무 강하면 실패
  • Mode Collapse 문제 - 다양성 상실 위험
  • 창조의 시작 - AI가 새로운 이미지 생성
  • Diffusion이 대세 - 하지만 GAN도 여전히 중요
