TOPIC 03

이미지는 몇 차원이야?

Q. 28×28 흑백 이미지는 어떻게 표현해?

Part 1. 차원이 뭐야?

이미지가 몇 차원이라는 게 무슨 뜻이야?

차원 = 데이터를 표현하는 데 필요한 축의 개수

쉽게 말해, 숫자를 배열할 때 몇 개의 방향이 필요한지야.

차원 이해하기: 0차원: → 숫자 하나 → 예: 5 1차원: → 일렬로 늘어선 숫자 (배열) → 예: [1, 2, 3, 4, 5] 2차원: → 표 형태 (행렬) → 예: [[1, 2, 3], [4, 5, 6], [7, 8, 9]] 3차원: → 여러 개의 표를 쌓아놓은 것 → 예: 책 여러 페이지

이해됐어?

차원이 높아질수록 더 복잡한 데이터를 표현할 수 있어.

Part 2. 흑백 이미지 = 2차원

28×28 흑백 이미지는 몇 차원이야?

2차원

가로(28) × 세로(28) = 2개의 축

28×28 흑백 이미지: Shape: (28, 28) ↑ ↑ 세로 가로 실제 데이터: [ [0, 12, 45, 78, ..., 255], ← 첫 번째 행 (28개 숫자) [5, 34, 67, 89, ..., 200], ← 두 번째 행 [10, 23, 56, 123, ..., 150], ... [0, 0, 0, 0, ..., 0] ← 28번째 행 ] → 28개 행, 각 행마다 28개 숫자 → 총 28 × 28 = 784개 숫자
근데 왜 (28, 28)이야? (784)가 아니고?

엄청 중요한 질문이야!

(784)로 만들 수도 있어. 이걸 "평탄화(Flatten)"라고 해.

하지만 공간 정보가 사라져.

비교: 2차원 (28, 28): [ [0, 12, 45], [5, 34, 67], [10, 23, 56] ] → "위쪽 왼쪽에 있는 픽셀" 같은 위치 정보 보존 1차원 (784): [0, 12, 45, 5, 34, 67, 10, 23, 56, ...] → 그냥 숫자 나열 → "어디에 있던 픽셀인지" 정보 손실

일반 신경망은 1차원으로 펼쳐서 처리하고,

CNN은 2차원 구조를 유지하면서 처리해.

그래서 CNN이 이미지에 훨씬 강한 거야.

Part 3. 컬러 이미지 = 3차원

그럼 컬러 이미지는?

3차원!

가로 × 세로 × 채널 = 3개의 축

224×224 컬러 이미지: Shape: (224, 224, 3) ↑ ↑ ↑ 세로 가로 채널(RGB) 실제 데이터: [ # R 채널 (224×224) [[255, 200, 150, ...], [240, 190, 140, ...], ...], # G 채널 (224×224) [[100, 120, 130, ...], [110, 125, 135, ...], ...], # B 채널 (224×224) [[50, 60, 70, ...], [55, 65, 75, ...], ...] ] → 224 × 224 × 3 = 150,528개 숫자
그럼 3차원을 어떻게 상상해야 돼?

책장을 생각해봐.

비유: 책 한 페이지 = 2차원 (가로 × 세로) 책 여러 페이지 = 3차원 (가로 × 세로 × 페이지 수) 컬러 이미지: → R, G, B 페이지가 3장 있는 책 → 각 페이지는 224×224 표 → 3개를 겹치면 완성된 컬러 이미지

Part 4. 프로그래밍에서는 어떻게 써?

실제 코드로 보여줘

Python과 NumPy를 사용한다면 이렇게 표현해.

import numpy as np # 흑백 이미지 (28×28) grayscale = np.array([[0, 128, 255], [64, 192, 32], [255, 0, 128]]) print(grayscale.shape) # (3, 3) # 컬러 이미지 (3×3×3) rgb = np.array([ # R 채널 [[255, 0, 0], [0, 0, 0], [0, 0, 0]], # G 채널 [[0, 0, 0], [0, 255, 0], [0, 0, 0]], # B 채널 [[0, 0, 0], [0, 0, 0], [0, 0, 255]] ]) print(rgb.shape) # (3, 3, 3)

.shape는 배열의 차원을 보여줘.

(28, 28) = 2차원

(224, 224, 3) = 3차원

Part 5. AI 학습할 때는 4차원?

AI 학습할 때는 이미지를 한 장만 넣어?

아니. 여러 장을 한꺼번에 넣어.

이걸 "배치(Batch)"라고 해.

배치 처리: 이미지 1장: (224, 224, 3) 이미지 32장 배치: (32, 224, 224, 3) ↑ ↑ ↑ ↑ 배치 세로 가로 채널 → 4차원! 실제 학습: Step 1: 32장씩 묶어서 입력 Step 2: 한꺼번에 처리 (병렬 처리) Step 3: 평균 손실 계산 Step 4: 가중치 업데이트
왜 한 장씩 안 하고 여러 장을 한꺼번에 해?

여러 이유가 있어:

1. GPU가 병렬 처리를 잘해 - 32장 한꺼번에 처리하는 게 32번 처리하는 것보다 빠름

2. 안정적인 학습 - 여러 이미지의 평균 오차로 업데이트하면 덜 흔들림

3. 메모리 효율 - 한 번에 로딩하면 I/O 횟수 감소

데이터 Shape 차원
숫자 하나 ( ) 0차원
숫자 배열 (784,) 1차원
흑백 이미지 (28, 28) 2차원
컬러 이미지 (224, 224, 3) 3차원
이미지 배치 (32, 224, 224, 3) 4차원

핵심 정리

  • 흑백 이미지 = 2차원 - (높이, 너비) 형태
  • 컬러 이미지 = 3차원 - (높이, 너비, 채널) 형태
  • 배치 = 4차원 - (배치크기, 높이, 너비, 채널)
  • 2차원 유지가 중요 - 공간 정보를 보존해야 패턴 인식 가능
  • CNN은 다차원 처리 - 구조를 유지하면서 학습
