TOPIC 08

맥스 풀링과 평균 풀링은 뭐가 달라?

Q. 제일 큰 값만 남기는 이유는?

Part 1. 두 가지 풀링 방법

풀링 방법이 여러 가지야?

주로 두 가지를 써.

맥스 풀링(Max Pooling)과 평균 풀링(Average Pooling)

같은 입력, 다른 결과: 입력 (2×2): [1 3] [5 6] 맥스 풀링: → 최댓값 선택 → 결과: 6 평균 풀링: → 평균 계산 → (1+3+5+6) / 4 = 3.75

맥스는 "가장 강한 신호"를 남기고,

평균은 "전체 요약"을 만들어.

Part 2. 맥스 풀링이 인기 있는 이유

왜 맥스 풀링을 더 많이 써?

이미지 특징 추출에 더 적합하거든.

맥스 풀링의 장점: 1. 특징 강조 → "엣지가 있다" 같은 강한 신호 보존 → 약한 노이즈는 무시 2. 희소성 (Sparsity) → 중요한 것만 전달 → 불필요한 정보 제거 3. 역전파 효율 → 그래디언트가 최댓값에만 흐름 → 학습이 더 sharp함 예시: [0, 0, 0, 8] → 맥스: 8 (특징 살아있음) [0, 0, 0, 8] → 평균: 2 (특징 희석됨)

"이 영역에 엣지가 있다!"는 신호를 최대한 보존하는 게 중요해.

맥스 풀링이 이걸 잘해.

Part 3. 평균 풀링은 언제 써?

그럼 평균 풀링은 쓸모없어?

아니. 특정 상황에서는 더 좋아.

상황 맥스 풀링 평균 풀링
특징 추출 ◯ 강한 신호 보존 △ 신호 희석
노이즈 제거 △ 노이즈도 살릴 수 있음 ◯ 평균으로 완화
위치 정보 △ 손실 큼 ◯ 더 보존됨
Global Pooling △ 덜 사용 ◯ 주로 사용
평균 풀링 사용 사례: 1. Global Average Pooling → 마지막 층에서 사용 → (7, 7, 512) → (1, 1, 512) → 완전 연결층 대신 2. 질감 분석 → 전체적인 패턴이 중요할 때 → 섬유 검사, 표면 분석 등 3. Segmentation → 픽셀 단위 예측 → 공간 정보 중요

Part 4. 실제 차이 비교

구체적 예시: 입력 특징 맵 (엣지 검출 후): [0 0 8 8] [0 0 9 7] [0 0 0 0] [0 0 0 0] 2×2 맥스 풀링: [0 9] [0 0] → 엣지 신호(9) 선명하게 유지 2×2 평균 풀링: [0 8] [0 0] → (0+0+8+9)/4 = 4.25 → 반올림 8 → 신호 약해짐
그럼 왜 평균을 써?

Global Average Pooling 때문이야.

Global Average Pooling (GAP): 전통적 방법: Conv → Flatten → FC(완전연결) → 분류 → 파라미터 엄청 많음 GAP 사용: Conv → Global Average Pooling → 분류 → 파라미터 거의 없음 예시: (7, 7, 512) 특징 맵 → 각 512개 채널의 7×7 값을 평균 → (1, 1, 512) = 512개 숫자 → 바로 분류에 사용 장점: → 과적합 방지 → 파라미터 대폭 감소 → 위치 불변성 강화

Part 5. 어떤 걸 선택해야 해?

그래서 어떤 걸 쓰는 게 좋아?

대부분은 맥스 풀링을 써.

선택 가이드: 중간 층 (특징 추출): → 맥스 풀링 (99% 사용) → 특징을 선명하게 유지 마지막 층 (분류 직전): → Global Average Pooling → FC layer 대신 특수한 경우: → L2 Pooling (거리 계산) → Stochastic Pooling (랜덤) → 하지만 거의 안 씀 실무 조언: → 일단 맥스 풀링 쓰고 → 성능 안 나오면 다른 거 실험 → 최신 모델은 Strided Conv로 대체 경향

AlexNet, VGG, ResNet 모두 맥스 풀링을 써.

검증된 방법이니까 기본으로 선택하면 돼.

핵심 정리

  • 맥스 풀링 = 최댓값 - 강한 신호 보존, 특징 강조
  • 평균 풀링 = 평균값 - 전체 요약, 노이즈 완화
  • 맥스가 일반적 - 중간 층에서는 거의 항상 맥스
  • GAP는 평균 - Global Average Pooling은 마지막 층
  • 용도에 따라 선택 - 특징 추출 vs 전체 요약
