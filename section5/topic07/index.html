TOPIC 07

풀링은 왜 해?

Q. 이미지를 축소하면 뭐가 좋은데?

Part 1. 풀링(Pooling)이 뭐야?

풀링이 뭐야?

이미지를 작게 만드는 연산이야.

여러 픽셀을 하나로 요약하는 거지.

비유: 고화질 사진을 썸네일로 만드는 것과 비슷해. 원본: 1920×1080 썸네일: 192×108 (10분의 1) → 크기는 줄었지만 무엇인지는 알 수 있음 → 세부 사항은 사라지지만 핵심은 유지

풀링도 똑같아.

크기는 줄이되, 중요한 정보는 유지하는 게 목표야.

Part 2. 어떻게 작동해?

구체적으로 어떻게 축소해?

작은 영역을 하나의 값으로 대표하는 거야.

2×2 맥스 풀링 예시: 입력 (4×4): [1 3 2 4] [5 6 7 8] [3 2 1 0] [1 2 3 4] 2×2 영역으로 나누기: [1 3] [2 4] [5 6] [7 8] [3 2] [1 0] [1 2] [3 4] 각 영역에서 최댓값 선택: 6 8 3 4 출력 (2×2): [6 8] [3 4] → 크기가 절반으로!

이해됐어?

2×2 풀링을 하면 가로, 세로 각각 절반이 돼.

→ 전체 픽셀 수는 4분의 1

Part 3. 왜 굳이 줄여?

이미지를 작게 만들면 뭐가 좋은데?

여러 가지 장점이 있어.

풀링의 장점: 1. 계산량 감소 → 픽셀 수가 1/4로 줄면 계산도 1/4 → 학습 속도 빨라짐 2. 파라미터 감소 → 다음 층의 입력 크기 감소 → 필요한 파라미터도 감소 3. 과적합 방지 → 세부 사항을 제거 → 일반화 성능 향상 4. 위치 불변성 → 약간의 위치 변화에 robust → "왼쪽 위" vs "약간 오른쪽 위" → 같은 특징으로 인식 5. 수용 영역 증가 → 다음 층이 더 넓은 영역을 볼 수 있음

핵심은 "중요한 것만 남기고 불필요한 건 버린다"야.

Part 4. 정보가 손실되지 않아?

그럼 정보가 사라지는 거 아니야? 나쁜 거 아냐?

적절한 정보 손실은 오히려 좋아.

비유: 책을 요약할 때를 생각해봐. 원문 (300페이지): → 모든 세부 사항 포함 → 읽는 데 시간 오래 걸림 → 핵심 파악 어려움 요약 (30페이지): → 중요한 내용만 → 빠르게 이해 → 핵심 파악 쉬움 풀링도 마찬가지: → 세부 픽셀 값은 사라짐 → 하지만 "엣지가 있다", "밝은 영역이다" 같은 핵심은 유지 → 오히려 노이즈가 제거돼서 더 강건함

실제로 풀링 없이 CNN을 만들면:

→ 과적합 심함

→ 계산량 너무 많음

→ 학습 어려움

적절한 손실이 오히려 성능을 높여.

Part 5. 실제 CNN에서는?

실제로는 어떻게 쓰여?

보통 합성곱 → 풀링을 반복해.

전형적인 CNN 구조: 입력: (224, 224, 3) Conv → ReLU → Pool → (224, 224, 64) → (112, 112, 64) Conv → ReLU → Pool → (112, 112, 128) → (56, 56, 128) Conv → ReLU → Pool → (56, 56, 256) → (28, 28, 256) Conv → ReLU → Pool → (28, 28, 512) → (14, 14, 512) Conv → ReLU → Pool → (14, 14, 512) → (7, 7, 512) Flatten → FC → (7×7×512 = 25,088) → 분류

보이지?

공간 크기(가로×세로)는 점점 줄고,

채널 수(깊이)는 점점 늘어나.

→ 넓고 얕은 정보 → 좁고 깊은 정보

Part 6. 요즘은 풀링 안 쓴다고?

최신 모델은 풀링을 안 쓴다는 얘기를 들었는데?

맞아. 요즘은 대안이 있어.

풀링의 대안: 1. Strided Convolution → 합성곱의 stride를 2로 설정 → 풀링 없이도 크기 축소 → 학습 가능해서 더 유연 2. Global Average Pooling → 마지막에만 사용 → 각 채널의 평균을 하나의 값으로 → 완전 연결층 대신 사용 3. Attention Mechanism → 중요한 부분에 집중 → 단순 축소가 아닌 선택적 처리 최신 트렌드: → ResNet: 풀링 최소화 → Vision Transformer: 풀링 없음 → EfficientNet: Strided Conv 선호

하지만 여전히 많은 모델이 풀링을 써.

특히 계산 효율이 중요할 때는 풀링이 유용해.

핵심 정리

  • 풀링 = 다운샘플링 - 이미지 크기를 줄이는 연산
  • 계산량 감소 - 픽셀 수가 줄어 학습 빨라짐
  • 과적합 방지 - 세부 사항 제거로 일반화 향상
  • 위치 불변성 - 약간의 이동에 robust
  • 적절한 정보 손실 - 노이즈 제거, 핵심 유지
  • 대안 존재 - Strided Conv, Global Pooling 등
