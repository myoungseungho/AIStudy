TOPIC 10

특징(Feature)이 뭐야?

Q. 사진을 숫자로 어떻게 표현해?

Part 1. AI가 보는 데이터

특징(Feature)이 뭐야? AI한테 사진을 보여주면 어떻게 이해하는 거야?

AI는 사진을 "숫자 덩어리"로 봐.

그 숫자들을 우리는 "특징"이라고 불러.

사람이 보는 것 vs AI가 보는 것: 사람: 🐱 고양이 사진 → "귀여운 고양이다!" → 귀가 뾰족, 수염, 동그란 눈... AI: [0.8, 0.3, 0.9, 0.1, ...] → 픽셀 값들의 배열 → 숫자로만 이해 고양이 사진 (100x100 픽셀): → 10,000개 숫자 → 각 픽셀이 RGB 3개 값 → 총 30,000개 숫자! → 이 30,000개 숫자가 "특징"

이해됐어?

특징 = AI가 데이터를 이해하기 위한 숫자 표현

Part 2. 여러 종류의 특징

모든 데이터를 숫자로 바꾼다고?

그래. 어떤 데이터든 숫자로 변환해야 AI가 이해해.

데이터별 특징 추출: 1. 이미지 원본: 고양이 사진 특징: [빨강값, 초록값, 파랑값, ...] × 픽셀 수 2. 텍스트 원본: "오늘 날씨가 좋아" 특징: - 단어 빈도: {"오늘": 1, "날씨": 1, "좋아": 1} - 숫자: [0, 1, 0, 0, 1, 0, 1, ...] 3. 음성 원본: 음성 파일 특징: - 주파수: [440Hz, 880Hz, ...] - 음량: [0.5, 0.7, 0.3, ...] - 시간: [0.1초, 0.2초, ...] 4. 표 데이터 (집 정보) 원본: 평수: 32평 위치: 강남 층수: 5층 특징: [32, 1, 5] (위치는 강남=1, 기타=0)

모든 데이터는 결국 "숫자 배열"로 변환돼.

Part 3. 좋은 특징 vs 나쁜 특징

특징이면 다 똑같은 거 아니야?

아니, 좋은 특징과 나쁜 특징이 있어.

고양이/개 구분 AI 나쁜 특징: - 사진 파일 크기 - 촬영 날짜 - 파일 이름 → 고양이/개와 관계없는 정보 애매한 특징: - 배경 색깔 - 사진 밝기 → 때로 도움되지만 본질 아님 좋은 특징: - 귀 모양 (뾰족/축 늘어짐) - 코 길이 - 수염 유무 → 고양이/개를 구분하는 핵심! 좋은 특징의 조건: 1. 문제와 관련있음 2. 구분력이 높음 3. 측정 가능함

특징 선택이 AI 성능을 좌우해.

쓸데없는 특징 넣으면 성능 오히려 나빠져.

Part 2. 특징 벡터

특징들을 어떻게 정리해?

특징 벡터(Feature Vector)로 만들어.

특징 벡터 예시: 집 데이터: 집1: 평수: 32평 방: 3개 화장실: 2개 층수: 5층 역 거리: 500m → 특징 벡터: [32, 3, 2, 5, 0.5] 집2: 평수: 25평 방: 2개 화장실: 1개 층수: 3층 역 거리: 200m → 특징 벡터: [25, 2, 1, 3, 0.2] AI 입장: "첫 번째 숫자가 크고, 다섯 번째 숫자가 작으면 비싸구나" → 패턴 학습

특징 벡터 = 데이터를 AI가 이해할 수 있는 형태로 만든 것

Part 5. 특징 스케일링

숫자 크기가 다르면 문제 안 돼?

맞아! 그래서 스케일링(Scaling)을 해.

스케일링 전: 평수: 32평 (10~50 범위) 역 거리: 500m (100~2000 범위) 층수: 5층 (1~20 범위) 문제: → 역 거리(500)가 평수(32)보다 훨씬 큼 → AI가 역 거리에만 집중 → 평수를 무시함 스케일링 후 (0~1로 정규화): 평수: (32-10)/(50-10) = 0.55 역 거리: (500-100)/(2000-100) = 0.21 층수: (5-1)/(20-1) = 0.21 → 모두 0~1 사이 → 공정하게 학습 가능 다른 방법 (표준화): 평균 0, 표준편차 1로 변환 → 통계적으로 공정

특징들의 스케일을 맞춰줘야 AI가 제대로 학습해.

Part 6. 차원의 저주

특징이 많으면 좋은 거 아냐?

아니, 너무 많으면 오히려 문제야.

이걸 차원의 저주(Curse of Dimensionality)라고 해.

차원의 저주: 데이터: 100개 특징 3개: → 충분히 학습 가능 → 패턴 찾기 쉬움 특징 1000개: → 데이터가 너무 "희박"함 → 100개 데이터로는 부족 → 과적합 쉬움 비유: 1차원 (선): 점 10개 → 빽빽 2차원 (면): 점 10개 → 여유 3차원 (공간): 점 10개 → 텅텅 10차원: 점 10개 → 거의 빈 공간 → 차원이 높아질수록 데이터가 희박해짐 → 패턴 찾기 어려워짐 해결책: 1. 차원 축소 (PCA) 2. 중요한 특징만 선택 3. 더 많은 데이터 수집

특징은 "적당히" 많아야 해. 너무 많으면 독이 돼.

핵심 정리

  • 특징 = 숫자 표현 - AI가 데이터를 이해하기 위한 숫자 배열
  • 모든 데이터 변환 - 이미지, 텍스트, 음성 모두 숫자로 변환
  • 좋은 특징 선택 - 문제와 관련있고 구분력 높은 특징
  • 특징 벡터 - 여러 특징을 하나의 배열로 정리
  • 스케일링 필수 - 특징들의 크기를 맞춰야 공정한 학습
  • 차원의 저주 - 특징이 너무 많으면 오히려 성능 저하
