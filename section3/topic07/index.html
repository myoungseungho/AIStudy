TOPIC 07

과적합이 뭐야?

Q. 훈련 데이터만 잘 맞추면 안 돼?

Part 1. 시험 문제만 달달 외운 학생

과적합(Overfitting)이 뭐야? 이름만 들어도 뭔가 안 좋은 거 같은데

훈련 데이터만 너무 완벽하게 맞춰서, 새로운 데이터는 못 맞추는 현상이야.

학교 시험으로 비유해볼게.

비유: 기출문제만 외운 학생 학생A (과적합): - 기출문제 100문제를 달달 외움 - 기출문제: 100점 - 새 문제: 40점 (응용 못함) 학생B (적절한 학습): - 기출문제로 개념 이해 - 기출문제: 90점 - 새 문제: 85점 (응용 가능) → 누가 더 잘 공부한 거야? → 당연히 학생B지!

이해됐어?

과적합 = 훈련 데이터를 외워버린 거야. 패턴을 배운 게 아니라.

Part 2. AI의 과적합 예시

AI도 그렇게 되는 거야?

그럼. 똑같이 일어나.

고양이/개 분류 AI 훈련 데이터: 100장 - 고양이 50장 (전부 실내에서 찍음) - 개 50장 (전부 야외에서 찍음) 과적합된 AI: "배경이 집이면 → 고양이" "배경이 풀밭이면 → 개" → 훈련 데이터: 100% 정확 → 테스트 데이터: - 야외 고양이 사진 → "개" (틀림!) - 실내 개 사진 → "고양이" (틀림!) → AI가 "고양이/개"를 배운 게 아니라 → "실내/야외"를 배워버림! 정상적인 AI: "귀가 뾰족하고 수염 있으면 → 고양이" "귀가 축 늘어지고 코가 길면 → 개" → 어떤 배경이든 잘 맞춤

과적합은 "훈련 데이터의 노이즈까지 학습"하는 거야.

진짜 패턴이 아니라 우연한 특징을 학습해버림.

Part 3. 과적합을 어떻게 알아채?

과적합됐는지 어떻게 알아?

훈련 성능과 테스트 성능의 차이를 보면 돼.

과적합 감지: 정상 케이스: 훈련 정확도: 92% 테스트 정확도: 90% → 차이: 2%p (괜찮음) 약간 과적합: 훈련 정확도: 95% 테스트 정확도: 85% → 차이: 10%p (경계 필요) 심각한 과적합: 훈련 정확도: 99% 테스트 정확도: 60% → 차이: 39%p (문제!) → 훈련 데이터 외운 것 과소적합: 훈련 정확도: 50% 테스트 정확도: 48% → 둘 다 낮음 → 제대로 학습 못함

훈련 성능만 보고 "잘됐다!"라고 착각하면 안 돼.

반드시 테스트 성능과 비교해야 해.

Part 4. 과적합은 왜 생겨?

과적합은 왜 일어나는 거야?

여러 원인이 있어.

과적합 원인: 1. 데이터가 너무 적음 → 데이터 10개로 학습 → 10개 특징을 전부 외워버림 2. 모델이 너무 복잡함 → 간단한 문제인데 너무 복잡한 AI 사용 → 필요 없는 세부사항까지 학습 3. 학습을 너무 오래 함 → 1만 번 반복 학습 → 노이즈까지 완벽하게 암기 4. 정규화를 안 함 → 제약 없이 자유롭게 학습 → 과하게 최적화됨

비유하면:

시험 문제 10개를 100번 반복해서 외우면

문제 번호만 봐도 답을 알게 되는 것처럼,

AI도 데이터를 너무 많이 보면 외워버려.

Part 5. 과적합을 막는 방법

그럼 과적합을 어떻게 막아?

여러 방법이 있어.

과적합 방지 방법: 1. 데이터 더 모으기 → 10개 → 1000개로 늘리기 → 외우기 어려워짐 2. 모델 단순화 → 너무 복잡한 모델 → 적당한 모델 → 필요한 것만 학습 3. 조기 종료 (Early Stopping) → 테스트 성능이 나빠지기 시작하면 멈춤 → 과하게 학습하는 걸 방지 4. 정규화 (Regularization) → "너무 복잡하게 하면 패널티" → 단순한 패턴을 찾도록 유도 5. 드롭아웃 (Dropout) → 학습할 때 일부러 뉴런 끄기 → 특정 뉴런에 의존 못하게 함 6. 데이터 증강 (Data Augmentation) → 이미지 회전, 반전, 자르기 → 다양한 버전 만들어서 학습

핵심은 "외우지 못하게" 하는 거야.

일부러 어렵게 만들어서, 본질적인 패턴만 배우게 하는 거지.

Part 6. 과소적합 vs 과적합 vs 적절

그럼 적당한 지점은 어떻게 찾아?

학습 곡선을 보면서 조절해야 해.

학습 곡선 (Learning Curve): 정확도 ↑ 100% | | 테스트 성능 (최적점!) 80% | ● | ● ● 60% | ● ● (과적합 시작) | ● ● 40% | ● ● | 0 +--●--●--●--●--●--●--●→ 학습 횟수 ↑ ↑ ↑ 과소적합 적절 과적합 과소적합 (Underfitting): → 둘 다 낮음 → 더 학습 필요 적절 (Sweet Spot): → 테스트 성능 최고 → 이 지점에서 멈춰야 함 과적합 (Overfitting): → 훈련은 올라가는데 테스트는 내려감 → 너무 많이 학습함

AI 학습의 핵심은 "적당한 지점"을 찾는 거야.

너무 적게도, 너무 많이도 학습하면 안 돼.

핵심 정리

  • 과적합 = 훈련 데이터 암기 - 패턴이 아닌 데이터 자체를 외움
  • 감지 방법 - 훈련과 테스트 성능 차이가 크면 과적합
  • 원인 - 데이터 부족, 모델 복잡, 과도한 학습
  • 방지 방법 - 데이터 증가, 모델 단순화, 조기 종료, 정규화
  • 적절한 균형 - 과소적합과 과적합 사이의 최적점 찾기
  • 테스트가 중요 - 훈련 성능만 보면 과적합을 못 찾음
