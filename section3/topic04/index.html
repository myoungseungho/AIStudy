TOPIC 04

비지도학습은 정답 없이 배워?

Q. 레이블 없이 어떻게 학습해?

Part 1. 정답 없이도 배울 수 있어

비지도학습(Unsupervised Learning)은 정답이 없다고? 그럼 어떻게 배워?

데이터 자체에서 패턴을 찾아내.

사람으로 비유하면 이렇게 배우는 거야.

비유: 책장 정리하기 지도학습 방식: 엄마: "이건 '소설'이야, 여기 꽂아" 엄마: "이건 '만화'야, 저기 꽂아" → 누가 일일이 카테고리를 알려줌 비지도학습 방식: 나 혼자 책 100권 보면서: "어? 이 책들은 그림이 많네 → 한 묶음" "이 책들은 글만 있고 두껍네 → 한 묶음" "이 책들은 얇고 사진이 많네 → 한 묶음" → 스스로 비슷한 것들을 묶음

이해됐어?

정답을 주지 않아도, 스스로 "비슷한 것끼리" 그룹을 만들어.

Part 2. 왜 정답 없이 배우는 걸 써?

지도학습이 더 정확하지 않아? 왜 굳이 비지도학습을 써?

좋은 질문이야. 현실적인 이유가 있어.

지도학습의 문제: 고양이/개 분류 AI 만들기 → 이미지 10만 장에 사람이 "고양이", "개" 레이블 달기 → 시간: 몇 달 → 비용: 수천만원 비지도학습의 장점: 이미지 10만 장 그냥 넣기 → AI가 스스로 "이런 특징끼리 비슷하네" 그룹 만듦 → 시간: 몇 시간~며칠 → 비용: 거의 0원 → 레이블 비용을 아낄 수 있음!

핵심은 "레이블링 비용"이야.

데이터는 많은데 레이블 달 돈이 없을 때 비지도학습을 써.

Part 3. 비지도학습으로 뭘 할 수 있어?

정답 없이 뭘 배울 수 있는데?

크게 두 가지를 할 수 있어.

1. 클러스터링 (Clustering) - 그룹 나누기 예시: 고객 데이터 10만 명 → AI가 자동으로 그룹 분류 → "10대 여성 그룹", "30대 남성 그룹" 등 뉴스 기사 1만 개 → AI가 자동으로 주제별 분류 → "정치", "경제", "스포츠" 등 (사람이 주제를 안 알려줘도 됨!) 2. 차원 축소 (Dimensionality Reduction) - 압축하기 예시: 이미지 1000x1000 픽셀 (100만 개 숫자) → 핵심 특징 100개로 압축 → 용량 줄이고 학습 빠르게 → "중요한 것만 남기기"

보이지?

정답은 모르지만, 데이터의 숨겨진 구조를 찾는 거야.

Part 4. 클러스터링은 어떻게 작동해?

그룹을 어떻게 나눠?

간단한 예시로 보여줄게.

K-Means 클러스터링 (가장 기본적인 방법) 데이터: 학생1: 키 180cm, 몸무게 70kg 학생2: 키 160cm, 몸무게 50kg 학생3: 키 175cm, 몸무게 68kg 학생4: 키 155cm, 몸무게 48kg ... 과정: 1. "3개 그룹으로 나눠줘" 라고 요청 2. AI가 랜덤하게 3개 중심점 선택 3. 각 학생을 가장 가까운 중심점 그룹으로 배정 4. 그룹의 평균으로 중심점 다시 계산 5. 3-4 반복 (안정될 때까지) 결과: 그룹1: 키 크고 무거운 학생들 그룹2: 키 작고 가벼운 학생들 그룹3: 중간 학생들 → 누가 "이 학생은 그룹1"이라고 안 알려줘도 → 스스로 비슷한 애들끼리 묶음!

이해됐어?

"거리가 가까운 애들끼리" 그룹을 만드는 거야.

Part 5. 비지도학습의 문제점은?

그럼 비지도학습이 더 좋은 거 아니야? 레이블도 필요 없고

아니, 큰 단점이 있어.

비지도학습의 한계: 1. 결과를 보장할 수 없음 지도학습: "고양이"라고 정확히 맞춤 비지도학습: "그룹A"를 만들긴 하는데, 그게 고양이인지는 몰라 2. 해석이 어려움 AI가 고객을 3개 그룹으로 나눴는데 → 그룹1이 뭘 의미하는지 사람이 봐야 알 수 있음 3. 얼마나 나눠야 하는지 모름 "몇 개 그룹으로 나눌까?" 를 사람이 정해줘야 함 → 2개? 5개? 10개? 정답이 없음 4. 품질 평가가 어려움 지도학습: "정확도 95%!" 비지도학습: "이게 잘된 건가?" (측정하기 어려움)

비지도학습은 "탐색용"이야.

데이터에 어떤 패턴이 있는지 대충 보려고 쓰지,

정확한 예측이 필요하면 결국 지도학습을 써야 해.

Part 6. 실제로는 어디에 써?

그래서 실생활에서는 어떻게 쓰여?

생각보다 많이 쓰여.

실전 비지도학습 사례: 고객 세분화 (마케팅) - 고객 100만 명 데이터 - 비지도학습으로 자동 그룹 분류 - "이 그룹한테는 이런 광고를" 전략 수립 추천 시스템 (넷플릭스, 유튜브) - 비슷한 영상끼리 그룹 묶기 - "이 영화 본 사람은 이것도 좋아할 것" 이상 탐지 (보안) - 정상 패턴 학습 - 이상한 패턴 발견하면 경고 - 해킹, 사기 거래 탐지 이미지 압축 - 고화질 사진을 용량 줄이면서도 품질 유지 - JPEG, PNG 같은 압축 알고리즘

보이지?

정답은 모르지만, 패턴을 찾아야 할 때 비지도학습을 써.

핵심 정리

  • 비지도학습 = 정답 없이 학습 - 레이블 없이 데이터에서 패턴 찾기
  • 왜 쓰나? - 레이블링 비용이 너무 비싸거나 불가능할 때
  • 클러스터링 - 비슷한 것들끼리 자동으로 그룹 나누기
  • 차원 축소 - 데이터를 압축하면서 핵심 특징만 남기기
  • 한계 - 정확도 보장 안 됨, 해석 어려움, 평가 어려움
  • 활용 - 고객 세분화, 추천 시스템, 이상 탐지, 데이터 탐색
