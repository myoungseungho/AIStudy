TOPIC 11

특징 공학은 왜 해?

Q. AI한테 좋은 특징을 골라줘야 해?

Part 1. 특징 공학이 뭐야?

특징 공학(Feature Engineering)이 뭔데? AI가 알아서 하는 거 아니야?

특징 공학 = 사람이 AI한테 좋은 특징을 만들어주는 것

AI가 알아서 다 하지 못해. 사람의 도움이 필요해.

비유: 요리 재료 손질 AI = 요리사 데이터 = 재료 옵션1: 재료 그대로 주기 → 생 당근, 껍질 있는 감자... → 요리사가 손질까지 해야 함 → 시간 오래 걸림, 맛 별로 옵션2: 재료 손질해서 주기 (특징 공학!) → 당근 채썰기, 감자 껍질 벗기기... → 요리사는 요리만 집중 → 빠르고, 맛있음 특징 공학도 똑같아: 원본 데이터 그대로 → AI가 고생 좋은 특징으로 가공 → AI가 쉽게 학습

특징 공학은 AI가 학습하기 쉽게 데이터를 가공하는 거야.

Part 2. 왜 특징 공학이 필요해?

딥러닝은 알아서 특징 찾는다며? 왜 사람이 해야 돼?

좋은 지적이야. 딥러닝은 자동으로 특징을 찾아.

하지만 전통적인 머신러닝은 사람이 도와줘야 해.

전통 머신러닝 (특징 공학 필요): 집값 예측 원본 데이터: - 주소: "서울시 강남구 역삼동 123-45" - 건축년도: 2015 문제: → AI가 주소 문자열로는 학습 못함 → 건축년도만으로는 정보 부족 특징 공학 후: - 강남구 여부: 1 (예) - 역과의 거리: 500m - 건물 나이: 2024-2015 = 9년 - 평당 가격: 5000만원 → AI가 학습 가능한 형태! 딥러닝 (특징 공학 덜 필요): 이미지 분류 → 원본 픽셀 그대로 넣어도 됨 → 딥러닝이 알아서 특징 추출 But! → 데이터 많이 필요 → 계산 비용 높음

데이터가 적거나, 표 데이터라면 특징 공학이 필수야.

Part 3. 특징 공학 기법들

특징 공학은 구체적으로 어떻게 하는 건데?

여러 기법이 있어. 하나씩 보여줄게.

1. 결측치 처리 원본: [10, 20, NaN, 40] → 평균으로 채우기: [10, 20, 23, 40] → 0으로 채우기: [10, 20, 0, 40] 2. 범주형 → 숫자 변환 원본: ["서울", "부산", "서울", "대구"] → One-Hot: [[1,0,0], [0,1,0], [1,0,0], [0,0,1]] → 레이블: [0, 1, 0, 2] 3. 새로운 특징 생성 원본: 키(170cm), 몸무게(70kg) → 생성: BMI = 70 / (1.7²) = 24.2 4. 구간화 원본: 나이 25, 34, 52, 67 → 구간: 20대, 30대, 50대, 60대 5. 로그 변환 원본: 소득 [100만, 500만, 5000만] → 로그: [2.0, 2.7, 3.7] (차이 완화) 6. 상호작용 특징 원본: 평수(32), 강남(1) → 생성: 평수×강남 = 32 (강남 평수 중요도↑)

도메인 지식이 중요해. 문제를 잘 알아야 좋은 특징을 만들어.

Part 4. 실전 예시: 타이타닉 생존 예측

실제 예시로 보여줄 수 있어?

유명한 타이타닉 데이터로 보여줄게.

타이타닉 생존 예측 원본 데이터: 이름: "Mr. John Smith" 나이: 35 성별: "male" 객실등급: 3 동반가족: 2 특징 공학 전 (그대로 사용): → 이름은 학습 불가 → 성별 문자열 처리 필요 → 성능: 70% 특징 공학 후: 1. 이름에서 호칭 추출 "Mr. John Smith" → "Mr" → Mr/Mrs/Miss에 따라 생존율 다름 2. 나이 구간화 35세 → "30대" → 어린이(18세 미만) 생존율 높음 3. 성별 숫자화 "male" → 0 "female" → 1 4. 가족 크기 생성 동반가족 2명 → 가족크기 3명 → 혼자거나 너무 많으면 생존율 낮음 5. 객실등급×성별 3등급×남성 → 생존율 매우 낮음 → 성능: 85%!

같은 데이터인데 특징 공학으로 15% 성능 향상!

Part 5. 특징 선택

특징을 많이 만들수록 좋은 거 아냐?

아니, 중요한 특징만 골라야 해.

이걸 특징 선택(Feature Selection)이라고 해.

특징 선택 이유: 특징 100개 → 10개로 줄이기 장점: 1. 학습 속도 빠름 2. 과적합 방지 3. 해석하기 쉬움 4. 차원의 저주 방지 방법1: 상관관계 확인 → 정답과 상관관계 높은 특징만 → 예: 집값과 평수 상관 0.8 (높음) → 예: 집값과 파일크기 상관 0.01 (낮음, 제거) 방법2: 재귀적 제거 → 하나씩 빼보면서 성능 확인 → 성능 떨어지면 중요한 특징 → 안 떨어지면 불필요한 특징 제거 방법3: 중요도 확인 → 모델이 어떤 특징을 많이 쓰는지 → 중요도 낮은 특징 제거

"적지만 강력한" 특징이 "많지만 약한" 특징보다 나아.

Part 6. 딥러닝 시대의 특징 공학

딥러닝 쓰면 특징 공학 안 해도 돼?

상황에 따라 다르지만, 여전히 도움이 돼.

딥러닝 시대: 자동 특징 추출: 이미지 → CNN → 자동으로 특징 추출 → 특징 공학 거의 불필요 But, 여전히 필요한 경우: 1. 표 데이터 → 엑셀 같은 데이터 → 특징 공학 필수 2. 데이터 부족 → 딥러닝은 데이터 많이 필요 → 적으면 특징 공학으로 보완 3. 도메인 지식 활용 → 사람만 아는 규칙 → "월요일은 수요 낮다" 같은 지식 4. 해석 가능성 → 딥러닝은 블랙박스 → 특징 공학하면 해석 쉬움 결론: 딥러닝이 많은 걸 자동화했지만, 특징 공학은 여전히 중요한 기술!

도구는 발전해도, 데이터를 이해하는 건 여전히 사람의 몫이야.

핵심 정리

  • 특징 공학 = 데이터 가공 - AI가 학습하기 좋은 형태로 변환
  • 전통 ML 필수 - 딥러닝 아닌 경우 특징 공학이 성능 좌우
  • 다양한 기법 - 결측치, 범주형 변환, 새 특징 생성 등
  • 도메인 지식 중요 - 문제를 잘 알아야 좋은 특징 만들어
  • 특징 선택 - 중요한 특징만 골라서 사용
  • 여전히 중요 - 딥러닝 시대에도 표 데이터에는 필수
