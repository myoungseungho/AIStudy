TOPIC 13

로지스틱 회귀는 왜 '회귀'야?

Q. 분류 문제인데 왜 회귀라고 불러?

Part 1. 이름이 헷갈려

로지스틱 회귀(Logistic Regression)는 '회귀'인데 분류를 한다고? 뭐가 잘못된 거 아냐?

이름은 회귀지만, 실제로는 분류 알고리즘이야.

역사적인 이유로 이름이 이렇게 됐어. 헷갈리지.

비유: 붕어빵 붕어빵 ← 붕어가 아니야, 빵이야! 로지스틱 회귀 ← 회귀가 아니야, 분류야! 실제 하는 일: 선형 회귀: 숫자 예측 (집값, 온도...) 로지스틱 회귀: 카테고리 분류 (스팸/정상, 합격/불합격...) → 완전 다른 문제를 풀어

왜 이런 이름이 붙었는지 설명해줄게.

Part 2. 확률로 바꾸기

그럼 로지스틱 회귀는 어떻게 작동해?

선형 회귀 + 확률 변환이야.

선형 회귀의 문제: 문제: 이메일 스팸 분류 선형 회귀로 시도: 이메일1: 예측 = 0.8 이메일2: 예측 = 1.2 ← 문제! 이메일3: 예측 = -0.3 ← 문제! → 스팸(1) 또는 정상(0)이어야 하는데 → 1.2나 -0.3 같은 값 나옴 → 분류에 못 써! 로지스틱 회귀의 해결: 1단계: 선형 회귀 z = wx + b z = 1.5 (값 제한 없음) 2단계: 시그모이드 함수 적용 확률 = 1 / (1 + e^(-z)) 확률 = 0.82 (0~1 사이) 3단계: 분류 확률 ≥ 0.5 → 스팸 확률 < 0.5 → 정상 → 0.82 ≥ 0.5 → 스팸!

핵심은 "시그모이드 함수"로 0~1 사이 확률로 변환하는 거야.

Part 3. 시그모이드 함수

시그모이드 함수가 뭐야?

어떤 숫자든 0~1 사이로 압축하는 마법 함수야.

시그모이드 함수: σ(x) = 1 / (1 + e^(-x)) 입력 → 출력: -10 → 0.00005 (거의 0) -2 → 0.12 -1 → 0.27 0 → 0.5 (중간) 1 → 0.73 2 → 0.88 10 → 0.99995 (거의 1) 모양: 1.0 | ________ | / 0.5 | / | / 0.0 |____ -10 0 10 → S자 모양 (Sigmoid = S자형) → 끝에서는 평평, 중간은 급격 활용: wx + b = 2.5 → σ(2.5) = 0.92 → "92% 확률로 스팸"

시그모이드는 "확률로 해석 가능한 숫자"를 만들어줘.

Part 4. 학습 과정

로지스틱 회귀는 어떻게 학습해?

선형 회귀랑 비슷한데, 손실 함수가 달라.

로지스틱 회귀 학습: 데이터: 이메일1: 특징=[5, 10], 정답=스팸(1) 이메일2: 특징=[2, 3], 정답=정상(0) 초기: w=[0, 0], b=0 Step 1: 이메일1 예측: z = 0×5 + 0×10 + 0 = 0 확률 = σ(0) = 0.5 정답: 1 (스팸) 오차: 큼! Step 2: w, b 조정 w=[0.3, 0.4], b=0.1 이메일1 예측: z = 0.3×5 + 0.4×10 + 0.1 = 5.6 확률 = σ(5.6) = 0.996 정답: 1 (스팸) 오차: 작아짐! ...반복... 최종: 이메일1 → 확률 0.99 → 스팸 ✓ 이메일2 → 확률 0.05 → 정상 ✓

선형 회귀처럼 경사하강법으로 w와 b를 학습해.

Part 5. 다중 클래스 분류

로지스틱 회귀는 2개만 분류할 수 있어?

기본은 2개(이진 분류)인데, 확장하면 3개 이상도 가능해.

이진 분류 (Binary): 스팸 vs 정상 → 확률 하나: P(스팸) = 0.8 → P(정상) = 1 - 0.8 = 0.2 다중 클래스 (Multi-class): 뉴스 분류: 정치/경제/스포츠 Softmax 함수 사용: P(정치) = 0.1 P(경제) = 0.7 ← 가장 높음 P(스포츠) = 0.2 합계 = 1.0 → "경제" 선택! 방법: One-vs-Rest: - 정치 vs (경제+스포츠) - 경제 vs (정치+스포츠) - 스포츠 vs (정치+경제) → 3개 모델 만들어서 투표

로지스틱 회귀는 2개 분류가 기본, Softmax로 확장 가능

Part 6. 언제 쓰고 언제 안 써?

로지스틱 회귀는 언제 쓰는 게 좋아?

장단점이 명확해.

로지스틱 회귀 장점: 1. 간단하고 빠름 → 학습 속도 빠름 → 예측 속도 빠름 2. 확률 제공 → "80% 확률로 스팸" → 확신도 알 수 있음 3. 해석 가능 → w=[2, 3] → "첫 번째 특징이 중요" → 의사결정 설명 쉬움 4. 과적합 적음 → 단순한 모델 → 일반화 잘됨 로지스틱 회귀 단점: 1. 선형 관계만 → 복잡한 패턴 못 잡음 → 곡선 경계 안 됨 2. 특징 공학 필요 → 좋은 특징 만들어줘야 함 3. 클래스 불균형에 약함 → 스팸 99%, 정상 1%면 성능 떨어짐 언제 쓰나: ✓ 빠른 프로토타입 ✓ 해석 필요할 때 ✓ 간단한 분류 문제 ✓ 베이스라인 언제 안 쓰나: ✗ 복잡한 패턴 ✗ 비선형 관계 ✗ 이미지/음성 같은 고차원

로지스틱 회귀는 "간단하고 빠른" 분류의 대표주자야.

핵심 정리

  • 이름은 회귀, 실제는 분류 - 역사적 이유로 이름이 이상함
  • 선형 회귀 + 시그모이드 - 확률로 변환해서 분류
  • 0~1 확률 - "80% 확률로 스팸" 같은 해석 가능
  • 이진 분류 기본 - Softmax로 다중 클래스 확장 가능
  • 간단하고 빠름 - 프로토타입, 베이스라인용으로 좋음
  • 선형 한계 - 복잡한 패턴은 못 잡음
