TOPIC 10

이미지 생성 AI는 어떻게 그림을 그려?

Q. 미드저니는 그림을 복사하는 거야?

Part 1. 복사가 아니라 "그럴듯하게" 만들기

Midjourney, DALL-E는 기존 그림을 복사하는 거야?

아니야!

"그럴듯한" 이미지를 새로 만들어내는 거야.

비유: 요리사 복사하는 것: → 사진 보고 똑같이 따라 그리기 → 포토샵 복붙 AI가 하는 것: → "빵 + 야채 + 고기 = 햄버거"라는 개념 학습 → 새로운 햄버거 사진 요청하면 → 학습한 개념 조합해서 새 이미지 생성 → 똑같은 햄버거는 아니지만 → 햄버거처럼 보이는 이미지

Part 2. 노이즈에서 시작해

구체적으로 어떻게 그림을 그려?

디퓨전(Diffusion) 방식: 노이즈를 점점 제거

Stable Diffusion, Midjourney의 작동 원리: 1단계: 랜덤 노이즈 → 완전히 지저분한 픽셀 (TV 잡음 같은) 2단계: "고양이" 프롬프트 입력 → AI가 "고양이는 이런 패턴"이라고 학습한 걸 기억 3단계: 노이즈 제거 시작 → 조금씩 고양이 형태로 변형 → Step 1/50: 여전히 노이즈 → Step 10/50: 뭔가 동물 같음 → Step 25/50: 귀가 보임 → Step 50/50: 선명한 고양이 → 마치 안개가 걷히듯 이미지가 드러남

왜 이렇게?

• 랜덤 시작 = 매번 다른 결과

• 점진적 생성 = 중간에 조정 가능

• 안정적 = 한 번에 만드는 것보다 품질 좋음

Part 3. 텍스트 → 이미지 연결

"빨간 사과"라고 입력하면 어떻게 빨간 사과가 나와?

CLIP 모델이 텍스트와 이미지를 연결해.

CLIP의 역할: 학습 과정: 1. 수백만 장의 이미지 + 설명 쌍 → (사과 사진, "빨간 사과") → (고양이 사진, "귀여운 고양이") 2. 텍스트와 이미지를 같은 공간에 매핑 → "빨간" = 빨간색 픽셀 패턴 → "사과" = 둥근 형태 + 꼭지 → 조합 학습 생성 과정: 너: "빨간 사과" CLIP: 1. "빨간" 분석 → 빨간색 영역 강조 2. "사과" 분석 → 둥근 형태 강조 3. 노이즈 제거 과정에서 이 패턴 강화 결과: 빨간색 둥근 사과 이미지

Part 4. 스타일 이해

"고흐 스타일로"라고 하면 어떻게 그 스타일로 그려?

AI가 수많은 고흐 그림에서 "패턴"을 배웠어.

스타일 학습: 고흐 스타일 = ? AI가 학습한 패턴: - 굵고 뚜렷한 붓터치 - 소용돌이치는 듯한 선 - 강렬한 원색 대비 - 두꺼운 질감 "고흐 스타일로 고양이" → 위 패턴을 고양이 생성에 적용 → 붓터치 효과 + 소용돌이 + 강렬한 색 "사실적으로" → 사진 같은 디테일 → 정확한 명암 → 부드러운 그라데이션 → 스타일도 결국 학습한 "패턴"이야

Part 5. 저작권 논란

그럼 기존 그림을 베끼는 거 아니야? 저작권은?

핫한 논란이야.

찬성 측: "AI는 패턴을 학습하지, 복사하지 않는다" → 사람도 고흐를 보고 배우잖아? → AI도 마찬가지로 학습 "새로운 창작물이다" → 기존 그림을 붙여넣는 게 아님 → 완전히 새로운 픽셀 배치 반대 측: "학습 데이터 출처가 문제" → 작가 허락 없이 그림 수집 → 무단 사용 "작가의 스타일 도용" → "Greg Rutkowski 스타일로" 생성 → 작가 고유 스타일 무단 사용 "일자리 위협" → 일러스트레이터, 디자이너 대체 현재 상황: → 법적으로 아직 불명확 → 각국에서 법안 논의 중 → AI 회사들, 옵트아웃 옵션 제공 시작

Part 6. 한계와 미래

그럼 완벽해? 한계는 없어?

여전히 한계 많아.

현재 한계: 1. 손 그리기 어려움 → 손가락 6개, 이상한 각도 → 손은 복잡해서 패턴 학습 어려움 2. 텍스트 표현 약함 → "HELLO" 써달라면 → "HELO" 같은 이상한 글자 → 텍스트는 패턴이 아니라 정확한 규칙 3. 물리 법칙 무시 → 중력 무시한 물체 → 원근감 이상함 4. 세밀한 요청 어려움 → "왼쪽 고양이는 검정, 오른쪽은 흰색" → 정확한 위치 제어 약함 개선 중: → ControlNet (구도 제어) → Inpainting (부분 수정) → 멀티모달 (이미지+텍스트 동시 입력) → 빠르게 발전 중

📌 핵심 정리

  • 복사가 아님 - 기존 그림을 복사하는 게 아니라 패턴 학습 후 새로 생성
  • 디퓨전 - 노이즈에서 시작해 점점 선명한 이미지로
  • 텍스트 연결 - CLIP이 "빨간 사과" 같은 설명을 이미지로 변환
  • 스타일 학습 - 고흐, 사실적 같은 스타일도 패턴으로 학습
  • 저작권 논란 - 학습 데이터 출처, 스타일 도용 문제 존재
  • 여전히 한계 - 손, 텍스트, 물리 법칙은 아직 약함
