TOPIC 07

AI는 어떻게 '학습'해?

Q. 컴퓨터한테 뭘 가르친다는 거야?

Part 1. 사람의 학습 vs AI의 학습

AI가 "학습한다"는 게 무슨 뜻이야? 사람처럼 공부해?

비슷한 듯 다른 거야. 먼저 사람의 학습부터 볼게:

사람이 "고양이"를 배우는 과정: 1. 엄마가 고양이를 가리키며 "고양이"라고 말함 2. 귀여운 동물을 보면 "고양이?"라고 물음 3. 엄마가 "응, 맞아" 또는 "아니야, 강아지야" 4. 여러 번 반복 5. 고양이의 특징을 자연스럽게 이해 → 수염, 귀 모양, 울음소리 등 AI가 "고양이"를 배우는 과정: 1. 고양이 사진 10,000장 준비 2. 각 사진에 "고양이" 라벨 붙임 3. AI에게 보여주면서 "이게 고양이야" 4. AI가 픽셀 패턴 분석 → "귀가 뾰족, 수염 있음, 이런 픽셀 배치 = 고양이" 5. 새 사진 보여주면 "고양이 확률 95%"

핵심 차이: 사람은 "의미"를 이해, AI는 "패턴"만 인식

Part 2. 학습의 3가지 방식

AI는 한 가지 방법으로만 배워?

크게 3가지 방식이 있어.

1. 지도 학습 (Supervised Learning) → 선생님이 정답을 알려주며 가르침 예시: 스팸 메일 필터 - 입력: 이메일 내용 - 정답: "스팸" 또는 "정상" - 과정: 1만 개 이메일 + 정답 보여주기 - 결과: 새 이메일 보면 스팸인지 판단 2. 비지도 학습 (Unsupervised Learning) → 정답 없이 혼자서 패턴 찾기 예시: 고객 분류 - 입력: 고객 구매 데이터 - 정답: 없음 - 과정: AI가 알아서 비슷한 고객끼리 그룹화 - 결과: "이 그룹은 명품 선호, 저 그룹은 저렴한 거 선호" 3. 강화 학습 (Reinforcement Learning) → 보상을 받으며 시행착오로 배우기 예시: 게임 AI - 입력: 게임 화면 - 정답: 없음 - 과정: 이기면 보상 +1, 지면 -1 - 결과: 수만 번 플레이하며 승률 높이는 전략 학습

Part 3. 가중치 조정이 핵심

그래서 구체적으로 어떻게 배우는 건데?

핵심은 "가중치(Weight)" 조정이야.

초간단 예시: 학점 예측 AI 입력: - 출석 점수 - 과제 점수 출력: - 학점 (A, B, C...) AI 내부: 학점 = (출석 × W1) + (과제 × W2) 처음: W1 = 0.5, W2 = 0.5 (랜덤) 예측: 80점 (실제: 90점) ← 틀림! 조정: "과제가 더 중요하네?" W1 = 0.3, W2 = 0.7 다시 예측: 88점 (실제: 90점) ← 거의 맞음! → 이런 식으로 W1, W2를 계속 조정 → 수천 번 반복 → 최적의 가중치 찾음

"학습"의 정체:

학습 = 가중치를 조금씩 조정하는 과정

GPT-4는 가중치가 수천억 개야. 전부 이런 식으로 조정됨.

Part 4. 오차를 줄여나가기

가중치를 어떻게 조정해? 랜덤으로?

랜덤 아니야!

오차(Error)를 계산해서 줄이는 방향으로 조정해.

학습 과정 (단계별): 1. 예측 AI: "이 이미지는 고양이야!" 2. 정답 확인 실제: 강아지 3. 오차 계산 오차 = |예측 - 정답| = 크다! 4. 가중치 조정 "고양이 쪽으로 너무 치우쳤네, 반대로 조정하자" → W1, W2, W3... 전부 조금씩 변경 5. 다시 예측 AI: "이번엔 강아지 같은데?" 6. 오차 계산 오차 = 작아짐! 7. 반복 오차가 충분히 작아질 때까지 계속 → 이게 "학습"이야. → 수학적으로는 "경사 하강법(Gradient Descent)"

Part 5. 데이터 많이 보여주기

한 번만 보여주면 안 돼?

안 돼!

같은 데이터를 수십~수백 번 반복해서 봐야 해.

학습 과정 (실전): 데이터: 고양이 사진 10,000장 1회차 (Epoch 1): - 10,000장 전부 보여줌 - 정확도: 60% 2회차 (Epoch 2): - 같은 10,000장 다시 보여줌 - 정확도: 75% 3회차 (Epoch 3): - 또 같은 10,000장 - 정확도: 85% ... 20회차 (Epoch 20): - 정확도: 95% - "충분히 배웠다" 판단, 중단 → Epoch = 전체 데이터를 한 바퀴 도는 것 → 보통 수십~수백 Epoch 반복
왜 계속 봐야 해?

사람도 마찬가지야.

수학 문제집을 한 번만 풀면 안 외워지잖아?

여러 번 풀어야 공식이 몸에 배지.

AI도 똑같아. 반복이 학습의 핵심.

Part 6. 과적합 문제

그럼 무한정 많이 보여주면 더 똑똑해지는 거야?

아니야, 너무 많이 보여주면 문제가 생겨.

과적합(Overfitting)이라고 해.

과적합 예시: 학생이 수학 문제집을 너무 많이 풀어서 → 문제와 답을 통째로 외움 → 새 문제는 못 풀음 AI도 똑같아: 훈련 데이터: "정확도 99%!" 새 데이터: "정확도 60%..." ← 외워버림 해결법: 1. 검증 데이터 따로 준비 → 학습할 때 안 보여줌 → 나중에 테스트용으로 사용 2. 조기 종료 (Early Stopping) → 검증 데이터 정확도가 떨어지기 시작하면 중단 3. 정규화 (Regularization) → 가중치가 너무 커지지 않게 제한

핵심:

학습은 "적당히"가 중요해.

너무 적게 = 덜 배움

너무 많이 = 외워버림

📌 핵심 정리

  • 학습 = 패턴 인식 - 데이터에서 패턴을 찾아내는 과정
  • 3가지 방식 - 지도, 비지도, 강화 학습
  • 가중치 조정 - 학습의 본질은 가중치를 최적화하는 것
  • 오차 줄이기 - 예측과 정답의 차이를 계속 줄여나감
  • 반복이 핵심 - 같은 데이터를 여러 번 봐야 학습됨
  • 과적합 주의 - 너무 많이 학습하면 외워버림
