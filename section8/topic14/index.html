TOPIC 14

프롬프트 엔지니어링이 뭐야?

Q. 질문을 잘하면 답이 달라져?

Part 1. 프롬프트 엔지니어링이 뭐야?

같은 질문인데 어떻게 물어보느냐에 따라 답이 달라지던데?

프롬프트 엔지니어링(Prompt Engineering)은 AI에게 효과적으로 지시하는 기술이야.

"어떻게 물어보느냐"가 결과를 완전히 바꿔.

비유: 검색 엔진 vs AI: 검색 엔진: - "날씨"만 쳐도 알아서 지역 날씨 보여줌 - 단어만으로도 OK AI (ChatGPT): - "날씨" → "날씨에 대해 뭘 알고 싶으세요?" - "서울 내일 날씨 알려줘" → 정확한 답변 → 자세할수록 좋은 답 -------------------------------------------------- 나쁜 프롬프트: "글 써줘" AI: "무슨 글을? 어떤 톤으로? 얼마나 길게?" 좋은 프롬프트: "블로그용으로 '초보자를 위한 파이썬 기초' 주제로 1000자 분량, 친근한 말투로, 코드 예시 3개 포함해서 써줘" AI: 딱 원하는 대로 작성

프롬프트 = AI와 대화하는 언어야.

잘 쓸수록 원하는 결과를 얻어.

Part 2. 기본 프롬프트 기법

어떻게 잘 물어봐?

몇 가지 핵심 기법이 있어.

1. 명확한 지시 (Clear Instructions) ❌ 나쁜 예: "요약해줘" ✅ 좋은 예: "다음 뉴스 기사를 3문장으로 요약해줘. 핵심 사실만, 의견은 빼고." -------------------------------------------------- 2. 역할 부여 (Role Playing) ❌ 나쁜 예: "영어 배우고 싶어" ✅ 좋은 예: "너는 20년 경력의 영어 선생님이야. 초등학생 수준에 맞춰 현재진행형을 설명해줘." -------------------------------------------------- 3. 예시 제공 (Few-Shot Learning) ❌ 나쁜 예: "감정 분석해줘" ✅ 좋은 예: "다음 예시처럼 감정을 분석해줘: '오늘 기분 좋아' → 긍정 '짜증나네' → 부정 '괜찮아' → 중립 이제 이걸 분석해: '그럭저럭 살만해'" -------------------------------------------------- 4. 단계별 요청 (Step-by-Step) ❌ 나쁜 예: "수학 문제 풀어줘" ✅ 좋은 예: "수학 문제를 단계별로 풀어줘: 1단계: 문제 이해 2단계: 풀이 방법 설명 3단계: 계산 과정 4단계: 최종 답" -------------------------------------------------- 5. 제약 조건 (Constraints) ❌ 나쁜 예: "이야기 써줘" ✅ 좋은 예: "공포 이야기를 써줘. - 분량: 500자 - 주인공: 10대 여학생 - 배경: 폐교 - 결말: 반전 있게 - 비속어 금지" -------------------------------------------------- 6. 형식 지정 (Format) ❌ 나쁜 예: "장단점 알려줘" ✅ 좋은 예: "장단점을 다음 형식으로 알려줘: ## 장점 - 장점 1 - 장점 2 ## 단점 - 단점 1 - 단점 2"

자세하고, 구체적이고, 명확할수록 좋아.

Part 3. 고급 프롬프트 기법

더 고급 기술도 있어?

전문가들이 쓰는 고급 기법들이 있어.

1. Chain-of-Thought (CoT, 생각의 연쇄) 일반 프롬프트: "23 × 47 = ?" AI: "1081" (가끔 틀림) CoT 프롬프트: "23 × 47을 단계별로 계산해줘. 각 단계의 이유도 설명해." AI: "23 × 40 = 920 23 × 7 = 161 920 + 161 = 1081" → 더 정확 -------------------------------------------------- 2. Self-Consistency (자기 일관성) 같은 질문을 여러 번 → 가장 많이 나온 답 선택 -------------------------------------------------- 3. Tree of Thoughts (생각의 나무) 문제를 여러 방향으로 탐색 → 최적 경로 선택 "이 문제를 3가지 다른 방법으로 접근해봐. 각 방법의 장단점을 비교하고, 최선을 추천해줘." -------------------------------------------------- 4. ReAct (Reasoning + Acting) "문제를 풀기 전에 먼저 계획을 세워. 그 다음 계획대로 실행하고, 중간중간 스스로 검증해." -------------------------------------------------- 5. Meta-Prompting (메타 프롬프팅) AI에게 프롬프트를 작성하게 함 "내가 '블로그 글 작성'을 요청할 거야. 나를 위한 최적의 프롬프트를 만들어줘." AI가 더 좋은 프롬프트 생성 -------------------------------------------------- 6. Negative Prompting (부정적 프롬프팅) "하지 마"를 명시 "시 써줘. 단, 사랑이나 계절 이야기는 빼고."

이런 기법들은 복잡한 문제에 유용해.

간단한 질문엔 과하니까 상황에 맞게 써.

Part 4. 프롬프트 엔지니어 직업

이게 직업이 돼?

맞아. 프롬프트 엔지니어라는 직업이 생겼어.

연봉이 엄청 높아.

프롬프트 엔지니어: [역할] - AI 성능 최대화하는 프롬프트 설계 - 기업 맞춤 프롬프트 라이브러리 구축 - AI 제품 성능 최적화 - 사용자 가이드 작성 [연봬] - 미국: $175,000 ~ $335,000 (약 2억~4억 원) - 한국: 5000만~1억 원 - 경력 없이도 고연봉 가능 [필요 역량] - 코딩 필수 아님 (있으면 좋음) - AI 작동 원리 이해 - 창의성, 문제 해결력 - 언어 능력 (한국어+영어) - 반복 테스트 끈기 [채용 기업] - OpenAI, Anthropic, Google - 삼성, 네이버, 카카오 - AI 스타트업 - 컨설팅 회사 -------------------------------------------------- 실제 채용 공고 (Anthropic, 2023): Prompt Engineer - 연봬: $250K - $335K - 경력: 무관 (신입 OK) - 요구사항: * AI 모델 이해 * 창의적 문제 해결 * 글쓰기 능력 * Python 기초 (선택)

"AI 시대의 새로운 직업"이야.

코딩 못 해도, 문과여도 가능해.

Part 5. 실전 활용 예시

실제로 어떻게 써?

상황별 실전 프롬프트를 보여줄게.

1. 업무 이메일 나쁜: "이메일 써줘" 좋은: "다음 상황으로 이메일을 써줘: - 수신: 김철수 부장님 - 목적: 프로젝트 마감일 3일 연장 요청 - 이유: 추가 데이터 검증 필요 - 톤: 정중하지만 자신감 있게 - 분량: 200자 이내" -------------------------------------------------- 2. 코드 작성 나쁜: "파이썬 코드 써줘" 좋은: "파이썬으로 CSV 파일을 읽어서 평균을 계산하는 함수를 만들어줘. - 에러 처리 포함 - 주석으로 설명 - pandas 라이브러리 사용 - 초보자도 이해할 수 있게" -------------------------------------------------- 3. 번역 나쁜: "번역해줘" 좋은: "다음 한국어를 영어로 번역해줘: - 대상: 미국 고객 - 톤: 비즈니스 격식 - 문화적 뉘앙스 고려 - 직역보다 의역 우선 [한국어 텍스트]" -------------------------------------------------- 4. 요약 나쁜: "요약해줘" 좋은: "다음 보고서를 임원 브리핑용으로 요약: - 3개의 핵심 요점만 - 각 요점당 1문장 - 숫자 데이터 포함 - 결론과 제안 명확히" -------------------------------------------------- 5. 브레인스토밍 나쁜: "아이디어 줘" 좋은: "20대 여성 타겟 화장품 브랜드 네이밍 아이디어 10개를 다음 기준으로: - 발음 쉬움 - 기억하기 쉬움 - 긍정적 이미지 - 한국어+영어 각 5개 - 각 이름마다 간단한 설명"

프롬프트 = 지시서라고 생각해.

직원에게 일 시킬 때처럼 자세히 설명하는 거야.

Part 6. 프롬프트 팁과 주의사항

주의할 점은 뭐야?

몇 가지 팁과 주의사항이 있어.

프롬프트 작성 팁: ✅ DO (해야 할 것): - 구체적으로 - 예시 제공 - 형식 지정 - 제약 조건 명시 - 단계 나눠서 - 역할 부여 ❌ DON'T (하지 말 것): - 너무 짧게 - 애매하게 - 한 번에 너무 많이 - 모순되는 지시 -------------------------------------------------- 흔한 실수: 1. 과도한 기대 "완벽한 소설 써줘" → AI는 도구일 뿐, 마법 아님 2. 맥락 부족 "그거 해줘" → AI는 이전 대화 기억 but 한계 있음 3. 사실 검증 안 함 AI는 틀린 정보도 자신있게 말함 → 항상 확인! 4. 윤리 무시 타인 사칭, 거짓 정보 생성 → 악용 금지 -------------------------------------------------- 효율 높이는 방법: 1. 프롬프트 라이브러리 구축 - 잘 작동한 프롬프트 저장 - 재사용, 수정 2. 반복 개선 - 한 번에 완벽할 수 없음 - 결과 보고 프롬프트 수정 3. A/B 테스트 - 여러 프롬프트 비교 - 최선 선택 4. 커뮤니티 활용 - Prompt Engineering 커뮤니티 - 다른 사람 프롬프트 참고

AI는 도구야. 책임은 사용자에게 있어.

프롬프트를 잘 쓰는 것도 중요하지만, 윤리적으로 사용하는 게 더 중요해.

📌 핵심 정리

  • 프롬프트 엔지니어링 - AI에게 효과적으로 지시하는 기술
  • 기본 기법 - 명확한 지시, 역할 부여, 예시 제공, 단계별 요청, 제약 조건, 형식 지정
  • 고급 기법 - CoT, Self-Consistency, Tree of Thoughts, ReAct, Meta-Prompting
  • 새 직업 - 프롬프트 엔지니어 (연봉 $175K~$335K, 코딩 필수 아님)
  • 실전 활용 - 이메일, 코드, 번역, 요약, 브레인스토밍 등
  • 팁 - 구체적으로, 예시 제공, 반복 개선, 사실 검증 필수
  • 주의 - AI 맹신 금지, 윤리적 사용, 책임은 사용자
