TOPIC 11

개인정보와 AI는 어떤 관계야?

Q. 내 데이터는 어디에 쓰이는 거야?

Part 1. AI는 내 데이터를 어떻게 쓸까?

AI가 나를 너무 잘 아는 것 같아. 내 정보를 다 보고 있는 거야?

맞아. AI는 엄청난 양의 개인 데이터를 사용해.

검색 기록, 위치, 구매 내역, 좋아요, 심지어 얼굴까지.

AI 성능 = 데이터 양이니까 기업들은 계속 수집하려 하지.

AI가 수집하는 데이터: [온라인 행동] - 검색어, 클릭한 링크 - 머문 시간, 스크롤 패턴 - 구매 내역, 장바구니 - 좋아요, 댓글, 공유 [위치 정보] - GPS 위치 (실시간) - 자주 가는 곳 (집, 회사) - 이동 패턴, 속도 [생체 정보] - 얼굴 (얼굴 인식) - 목소리 (음성 비서) - 지문 (스마트폰) - 걸음걸이 (걸음 수) [소셜 정보] - 친구 관계 - 대화 내용 (메신저) - 사진 속 사람들 [기기 정보] - 어떤 앱 설치 - 언제 사용하는지 - 배터리, 저장공간 → 종합하면 "너"에 대한 프로필 완성

이 데이터로 AI는 너를 예측해.

뭘 살지, 어디 갈지, 누구와 만날지, 심지어 기분까지.

Part 2. 데이터 수집의 문제점

그게 뭐가 문제야? 난 숨길 거 없는데?

"숨길 게 없다"가 문제가 아니야.

프라이버시는 기본 권리야.

프라이버시 침해 사례: 1. 동의 없는 수집 - 앱이 몰래 위치 추적 - 카메라, 마이크 무단 접근 - 쿠키로 웹 전체 추적 2. 과도한 수집 - 손전등 앱이 연락처 요구 - 필요 이상의 데이터 수집 - 삭제해도 서버에 남음 3. 제3자 공유 - 내 데이터를 다른 회사에 팔기 - 광고 회사에 프로필 제공 - 정부 요청으로 제공 4. 유출 사고 - 해킹으로 데이터 유출 - 페이스북 8700만 명 유출 (2018) - 카카오톡 1천만 명 유출 (2023) 5. 악용 - 스토킹, 사생활 침해 - 신원 도용 - 타겟 광고, 조작 6. 감시 - 중국 사회 신용 점수 - 얼굴 인식으로 실시간 추적 - 행동 패턴 분석

"지금"은 문제없어도, 나중에 악용될 수 있어.

정권이 바뀌거나, 법이 바뀌면?

과거 데이터가 무기가 될 수 있어.

Part 3. 프라이버시 vs AI 성능 딜레마

그럼 데이터를 안 주면 돼?

문제는 데이터를 안 주면 AI가 작동 안 해.

이게 딜레마야.

Trade-off (트레이드오프): 더 많은 데이터 → 더 좋은 AI 더 적은 데이터 → 프라이버시 보호 -------------------------------------------------- 예시 1: 넷플릭스 데이터 많이 줄 때: - 시청 기록, 평점, 언제 보는지 → 완벽한 추천, 정확한 취향 파악 데이터 안 줄 때: - 로그인도 안 함 → 추천 X, 인기작만 봄 -------------------------------------------------- 예시 2: 구글 지도 데이터 많이 줄 때: - 위치 항상 공유 → 실시간 교통, 도착 시간 정확, 맞춤 추천 데이터 안 줄 때: - 위치 끔 → 기본 지도만, 실시간 정보 없음 -------------------------------------------------- 예시 3: 의료 AI 데이터 많이 줄 때: - 병력, 유전자, 생활습관 → 정확한 진단, 맞춤 치료 데이터 안 줄 때: - 최소 정보만 → 일반적 진단, 정확도 낮음

완벽한 균형은 없어.

각자 어디까지 공유할지 선택해야 해.

Part 4. 프라이버시 보호 기술

둘 다 가질 수는 없어? AI 성능도 유지하면서 프라이버시도 보호?

연구자들이 노력하고 있어. 몇 가지 기술이 있지.

프라이버시 보호 AI 기술: 1. Differential Privacy (차등 프라이버시) - 데이터에 "노이즈" 추가 - 전체 통계는 유지, 개인 식별 불가 - Apple이 사용 (키보드 학습) → 전체 패턴은 배우되, 개인 정보는 안 드러남 2. Federated Learning (연합 학습) - 데이터를 서버로 안 보냄 - 각 기기에서 학습 - 모델만 공유 → Google 키보드가 사용 3. Homomorphic Encryption (동형 암호) - 데이터를 암호화한 채로 연산 - 서버도 내용 못 봄 → 아직 느려서 실용화 어려움 4. Synthetic Data (합성 데이터) - 가짜 데이터 생성 - 통계적 특성은 유사 - 실제 사람 데이터 아님 → 테스트, 학습에 사용 5. Data Minimization (최소 수집) - 필요한 것만 수집 - 빨리 삭제 → GDPR 원칙 6. On-Device Processing (기기 내 처리) - 서버로 안 보내고 폰에서 처리 - Apple Face ID (얼굴 데이터 폰에만) → 유출 위험 감소

이런 기술들이 어느 정도 균형을 제공해.

완벽하진 않지만, 발전하고 있어.

Part 5. 법과 규제

법으로 보호받을 수 있어?

많은 나라가 프라이버시 법을 만들었어.

법/규제 지역 주요 내용
GDPR EU 동의 필수, 삭제권, 이동권, 설명 권리
CCPA 미국 캘리포니아 데이터 판매 거부, 삭제 요청 가능
개인정보보호법 한국 동의, 목적 제한, 최소 수집
신용정보법 한국 가명 처리, MyData (데이터 이동권)
GDPR 주요 권리: 1. 알 권리 (Right to be Informed) - 어떤 데이터를 어떻게 쓰는지 알려줘야 함 2. 접근권 (Right of Access) - 내 데이터 복사본 요청 가능 3. 정정권 (Right to Rectification) - 잘못된 데이터 수정 요구 4. 삭제권 (Right to Erasure, "잊힐 권리") - 내 데이터 삭제 요청 - 구글 검색 결과에서 제거 가능 5. 이동권 (Right to Data Portability) - 내 데이터를 다른 서비스로 이동 6. 거부권 (Right to Object) - 프로파일링, 자동화 거부 7. 설명 권리 - AI 자동 결정에 대한 설명 요구 위반 시: 최대 2천만 유로 or 매출 4% 벌금

권리가 있다는 걸 아는 게 중요해.

모르면 행사할 수 없으니까.

Part 6. 우리가 할 수 있는 것

나는 뭘 할 수 있어?

많은 걸 할 수 있어!

프라이버시 보호 실천법: [기본 설정] - 앱 권한 최소화 (위치, 카메라, 마이크) - 불필요한 앱 삭제 - 자동 업데이트로 보안 패치 [계정 관리] - 강력한 비밀번호 (각 사이트마다 다르게) - 2단계 인증 (2FA) 활성화 - 주기적으로 로그인 기록 확인 [웹 브라우징] - 시크릿 모드 사용 - 쿠키 주기적 삭제 - 추적 차단 (Ghostery, uBlock Origin) - VPN 사용 (공공 Wi-Fi에서) [소셜 미디어] - 공개 범위 설정 (친구만) - 위치 태그 끄기 - 민감한 정보 올리지 않기 - 주기적으로 과거 게시물 삭제 [AI 서비스] - 데이터 수집 최소화 설정 - 개인화 끄기 (정확도 떨어지지만 프라이버시 ↑) - 주기적으로 활동 기록 삭제 [이메일/메시지] - 의심스러운 링크 클릭 X - 개인정보 요구하는 이메일 무시 - 중요한 건 암호화 (Signal, ProtonMail) [구글/애플 설정] - 위치 기록 끄기 - 광고 개인화 끄기 - 활동 기록 자동 삭제 설정 (3개월) [정기 점검] - 3개월마다 앱 권한 검토 - 사용 안 하는 계정 삭제 - 데이터 유출 확인 (haveibeenpwned.com)

100% 프라이버시는 불가능해.

하지만 리스크를 줄일 수는 있어.

편리함과 프라이버시 사이에서 자신만의 균형을 찾는 거야.

📌 핵심 정리

  • AI와 데이터 - AI는 개인 데이터를 대량 수집·분석 (행동, 위치, 생체 정보)
  • 문제점 - 동의 없는 수집, 과도한 수집, 유출, 악용, 감시
  • 딜레마 - 더 많은 데이터 = 더 좋은 AI, 프라이버시 보호와 트레이드오프
  • 보호 기술 - 차등 프라이버시, 연합 학습, 기기 내 처리, 합성 데이터
  • 법적 권리 - GDPR (삭제권, 이동권, 설명권), 한국 개인정보보호법
  • 실천법 - 권한 최소화, 설정 점검, 추적 차단, 주기적 삭제
  • 균형 - 편리함 vs 프라이버시, 각자 선택이 필요
