TOPIC 15

AI 개발자가 되려면?

Q. 지금 뭐부터 시작해야 해?

Part 1. AI 개발자가 뭐야?

AI 개발자는 정확히 뭘 하는 사람이야?

AI 개발자는 넓은 개념이야. 여러 종류가 있어.

직무 하는 일 필요 역량
ML 엔지니어 모델 학습, 데이터 처리, 파이프라인 구축 Python, TensorFlow, PyTorch, 수학
AI 연구자 새로운 알고리즘 개발, 논문 작성 수학, 통계, 논문 읽기/쓰기, 박사
데이터 과학자 데이터 분석, 인사이트 도출, 예측 모델 통계, Python, SQL, 시각화
MLOps 엔지니어 모델 배포, 모니터링, 인프라 DevOps, Cloud, Docker, 쿠버네티스
AI 제품 매니저 AI 제품 기획, 전략, 팀 조율 AI 이해, 비즈니스, 커뮤니케이션
프롬프트 엔지니어 AI 최적 활용, 프롬프트 설계 AI 이해, 언어 능력, 창의성

자신의 강점과 관심사에 맞는 분야를 선택해.

수학 좋아하면 연구자, 비즈니스 좋아하면 PM, 코딩 좋아하면 엔지니어.

Part 2. 학습 로드맵 (초보자)

완전 초보인데 뭐부터 해야 해?

단계별로 천천히 가자.

한 번에 다 하려면 지쳐.

AI 개발자 로드맵 (0→1): [1단계: 기초 프로그래밍 (1~3개월)] - Python 기초 * 변수, 함수, 조건문, 반복문 * 리스트, 딕셔너리 * 추천: 생활코딩, 노마드코더, 점프 투 파이썬 - Jupyter Notebook 사용법 - Git 기초 [2단계: 수학/통계 (1~2개월)] - 선형대수 기초 (벡터, 행렬) - 확률/통계 기초 (평균, 분산, 확률분포) - 미분 기초 (편미분, 경사하강법) - 추천: 3Blue1Brown (YouTube), Khan Academy [3단계: 데이터 처리 (1~2개월)] - NumPy (배열 연산) - Pandas (데이터프레임) - Matplotlib, Seaborn (시각화) - 실습: Kaggle 데이터셋 [4단계: 머신러닝 기초 (2~3개월)] - Scikit-learn 라이브러리 - 회귀, 분류 알고리즘 - 모델 평가 (정확도, F1 Score) - 추천: Andrew Ng Coursera 강의 [5단계: 딥러닝 기초 (2~3개월)] - TensorFlow or PyTorch - 신경망 기본 개념 - CNN (이미지), RNN (시계열) - 실습: MNIST, CIFAR-10 [6단계: 프로젝트 (3개월~) - Kaggle 대회 참여 - 개인 프로젝트 (GitHub에 공개) - 블로그에 정리 - 포트폴리오 구축 -------------------------------------------------- 총 기간: 10개월 ~ 1년 (하루 2~3시간 기준)

조급하게 생각하지 마.

1년이면 충분히 기초를 다질 수 있어.

Part 3. 추천 학습 자료

어디서 배워?

무료와 유료 자료 모두 많아.

무료 학습 자료: [온라인 강의] - Coursera: Andrew Ng의 Machine Learning - Fast.ai: Practical Deep Learning (실전 중심) - YouTube: 3Blue1Brown (시각적 설명) - 모두의 딥러닝 (한국어) [책] - 파이썬 머신러닝 완벽 가이드 (권철민) - 밑바닥부터 시작하는 딥러닝 (사이토 고키) - Hands-On Machine Learning (번역: 핸즈온 머신러닝) [실습 플랫폼] - Kaggle: 데이터셋, 대회, 노트북 - Google Colab: 무료 GPU - Hugging Face: Pre-trained 모델 [커뮤니티] - Reddit: r/MachineLearning - Stack Overflow - GitHub (오픈소스 코드 읽기) -------------------------------------------------- 유료 자료 (선택): - Udemy: 실습 중심 강의 ($10~$20) - Coursera Specialization ($49/월) - DataCamp, Udacity Nanodegree (비쌈) -------------------------------------------------- 한국어 자료: - 모두를 위한 딥러닝 (김성훈 교수, KAIST) - 부스트코스: 머신러닝/딥러닝 과정 - 네이버 AI Lab 블로그 - 카카오 AI Report

무료 자료만으로도 충분해.

중요한 건 돈이 아니라 꾸준함이야.

Part 4. 실전 프로젝트 아이디어

뭘 만들어봐야 해?

포트폴리오용 프로젝트 아이디어를 줄게.

초급 프로젝트: 1. 영화 추천 시스템 - 데이터: MovieLens 데이터셋 - 기법: 협업 필터링 - 난이도: ★☆☆☆☆ 2. 손글씨 숫자 인식 - 데이터: MNIST - 기법: CNN - 난이도: ★★☆☆☆ 3. 스팸 메일 분류 - 데이터: SMS Spam Collection - 기법: 나이브 베이즈, Scikit-learn - 난이도: ★☆☆☆☆ -------------------------------------------------- 중급 프로젝트: 4. 고양이/개 분류 - 데이터: Dogs vs Cats (Kaggle) - 기법: Transfer Learning (ResNet, VGG) - 난이도: ★★★☆☆ 5. 감정 분석 (Sentiment Analysis) - 데이터: 영화 리뷰, 트위터 - 기법: RNN, LSTM, BERT - 난이도: ★★★☆☆ 6. 주식 가격 예측 - 데이터: Yahoo Finance API - 기법: LSTM, Time Series - 난이도: ★★★☆☆ -------------------------------------------------- 고급 프로젝트: 7. 챗봇 만들기 - 데이터: 대화 데이터셋 - 기법: Transformer, GPT Fine-tuning - 난이도: ★★★★☆ 8. 객체 탐지 (Object Detection) - 데이터: COCO, Pascal VOC - 기법: YOLO, Faster R-CNN - 난이도: ★★★★☆ 9. 이미지 생성 (GAN) - 데이터: CelebA (얼굴 이미지) - 기법: DCGAN, StyleGAN - 난이도: ★★★★★ -------------------------------------------------- 팁: - 처음엔 튜토리얼 따라하기 - 그 다음 내 아이디어로 변형 - GitHub에 코드 공개 - README에 자세히 설명 - 블로그에 과정 정리

완벽한 프로젝트보다 "완성한" 프로젝트가 중요해.

작더라도 끝까지 해보는 게 실력 향상의 지름길이야.

Part 5. 취업 준비

취업은 어떻게 준비해?

포트폴리오가 가장 중요해.

취업 준비 체크리스트: [포트폴리오] ✅ GitHub 프로젝트 3~5개 ✅ Kaggle 참여 (메달 따면 더 좋음) ✅ 블로그 (학습 과정, 프로젝트 설명) ✅ 논문 읽고 구현 (선택) [기술 역량] ✅ Python 능숙 ✅ TensorFlow or PyTorch ✅ 데이터 전처리 (Pandas, NumPy) ✅ SQL 기초 ✅ Git 사용법 [이론] ✅ ML 알고리즘 이해 ✅ 딥러닝 기초 (CNN, RNN, Transformer) ✅ 수학 (선형대수, 확률/통계) [코딩 테스트] ✅ 알고리즘 기초 (백준, 프로그래머스) ✅ Python 구현 능력 [소프트 스킬] ✅ 협업 경험 (팀 프로젝트) ✅ 문제 해결 능력 ✅ 커뮤니케이션 -------------------------------------------------- 이력서 팁: 1. 프로젝트 중심으로 작성 - 뭘 만들었나 - 어떤 기술 썼나 - 결과가 어떻게 됐나 - 수치로 표현 (정확도 10% 향상 등) 2. GitHub 링크 필수 3. 블로그 링크 (있으면 가산점) 4. Kaggle 프로필 (메달 있으면) 5. 간결하게 (2페이지 이내) -------------------------------------------------- 면접 준비: - 프로젝트 깊게 파고들기 - "왜 이 모델을 선택했나요?" - "어떤 어려움이 있었고 어떻게 해결했나요?" - 기술 질문 (과적합, 정규화, 활성화 함수 등) - 라이브 코딩 대비

학벌이나 전공보다 실력이 중요해.

비전공자도 포트폴리오만 탄탄하면 충분히 가능해.

Part 6. 현실적인 조언

솔직히 말해줘. 쉬워?

솔직히 말하면, 쉽지 않아.

하지만 불가능하지도 않아.

현실 체크: [어려운 점] - 수학이 생각보다 많이 필요 (특히 연구) - 계속 공부해야 함 (기술 빠르게 변함) - 경쟁 치열 (특히 대기업, 빅테크) - 초반 진입 장벽 높음 [하지만...] ✅ 수요가 엄청 많음 ✅ 연봉 높음 (신입도 4000만~6000만) ✅ 원격 근무 가능 ✅ 흥미로움 (지루하지 않음) ✅ 미래 전망 밝음 -------------------------------------------------- 성공하는 사람 특징: 1. 꾸준함 - 하루 2시간이라도 매일 - 1년 후 엄청난 차이 2. 호기심 - "왜?"를 계속 물음 - 논문 읽는 걸 즐김 3. 끈기 - 막혀도 포기 안 함 - 구글링, 커뮤니티 질문 4. 실습 중심 - 이론만 공부 X - 직접 코드 작성 5. 공유 - 배운 것 블로그 정리 - 오픈소스 기여 -------------------------------------------------- 흔한 실수: ❌ 모든 걸 완벽하게 배우려 함 → 계속 공부만, 프로젝트 안 함 ❌ 튜토리얼 지옥 → 강의만 계속 들음 ❌ 수학에 너무 집착 → 실전에서는 라이브러리 씀 ❌ 혼자만 공부 → 커뮤니티, 스터디 활용 안 함 -------------------------------------------------- 추천 마인드셋: ✅ 완벽주의 버리기 ✅ 작게 시작하기 ✅ 실패를 배움으로 ✅ 커뮤니티 활용 ✅ 장기적 관점 (1~2년)

AI 개발자는 마라톤이야. 단거리가 아니야.

조급해하지 말고, 꾸준히 한 걸음씩.

1년 후의 너는 지금과 완전히 다를 거야.
