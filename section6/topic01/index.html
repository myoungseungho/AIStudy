TOPIC 01

컴퓨터는 문장을 어떻게 이해해?

Q. 단어를 숫자로 바꾼다고?

Part 1. 컴퓨터는 문자를 이해할 수 없어

컴퓨터가 "안녕하세요"를 이해한다는 게 뭔 뜻이야?

사실 컴퓨터는 "안녕하세요"를 이해 못 해.

왜냐하면 컴퓨터는 오직 숫자만 다룰 수 있거든.

그럼 ChatGPT는 어떻게 대답하는 거야?

핵심은 이거야:

문자를 숫자로 변환한 다음 처리해.

마치 번역기처럼, 사람 언어 → 컴퓨터 언어(숫자) → 사람 언어로 변환하는 거지.

비유: 외국인과 대화하는 과정: 1. 한국어로 말한다 → "안녕하세요" 2. 통역사가 영어로 번역 → "Hello" 3. 외국인이 이해하고 대답 → "Nice to meet you" 4. 통역사가 한국어로 번역 → "만나서 반가워요" AI와 대화하는 과정: 1. 사람이 문자로 입력 → "안녕하세요" 2. AI가 숫자로 변환 → [245, 789, 1024] 3. AI가 숫자로 계산해서 답 생성 → [567, 123, 890] 4. AI가 다시 문자로 변환 → "안녕하세요! 무엇을 도와드릴까요?"

Part 2. 왜 숫자로 바꿔야 할까?

문자 그대로 처리하면 안 돼?

안 돼. 이유가 있어.

문제 설명
계산 불가 "강아지" + "고양이" = ? → 의미 없음
비교 불가 "강아지"가 "고양이"보다 큰가? → 비교 불가
유사도 측정 불가 "강아지"와 "개"가 얼마나 비슷한가? → 모름
패턴 학습 불가 딥러닝 모델은 숫자 연산으로 학습함

하지만 숫자로 바꾸면?

숫자로 변환하면 가능해지는 것들: "강아지" → [0.2, 0.8, 0.3] "개" → [0.3, 0.7, 0.4] "고양이" → [0.9, 0.1, 0.8] → 유사도 계산 가능 "강아지"와 "개"의 거리: 0.14 (매우 가까움) "강아지"와 "고양이"의 거리: 0.89 (좀 멈) → 연산 가능 "강아지" + "고양이" - "개" ≈ "고양이과 동물" → 패턴 학습 가능 [0.2, 0.8, 0.3] 근처에는 동물 관련 단어가 많다는 패턴 발견

Part 3. 그럼 어떻게 숫자로 바꾸는 거야?

단어를 어떻게 숫자로 만들어?

여러 방법이 있어. 가장 기본적인 것부터 보자.

방법 1. 단순히 번호 매기기

가장 단순한 방법:

사전 만들기: { "안녕": 1, "하세요": 2, "저는": 3, "AI": 4, "입니다": 5 } 문장: "안녕하세요 저는 AI입니다" → 숫자로 변환: [1, 2, 3, 4, 5]

문제: 이건 너무 단순해.

1, 2, 3, 4, 5라는 숫자만으로는 단어의 의미를 전혀 담지 못해.

"안녕"이 1이고 "AI"가 4라는 게 무슨 의미가 있어?

방법 2. 의미를 담은 벡터로 만들기

그래서 실제로는 벡터(여러 숫자의 리스트)로 표현해.

의미를 담은 벡터: "강아지" → [0.2, 0.8, 0.3, 0.9, 0.1] ↓ ↓ ↓ ↓ ↓ 귀여움 동물성 크기 친근함 야생성 "사자" → [0.1, 0.9, 0.9, 0.2, 0.8] ↓ ↓ ↓ ↓ ↓ 귀여움 동물성 크기 친근함 야생성 → "강아지"는 귀여움(0.2), 동물성(0.8), 작음(0.3), 친근함(0.9), 야생아님(0.1) → "사자"는 덜 귀여움(0.1), 동물(0.9), 큼(0.9), 덜 친근함(0.2), 야생(0.8)

물론 실제로는 이렇게 사람이 직접 정의하지 않아.

AI가 수백만 개의 문장을 읽으면서 자동으로 학습해.

Part 4. 실제로는 어떻게 쓰는 거야?

ChatGPT는 이걸 어떻게 쓰는 건데?

전체 과정을 보여줄게.

ChatGPT 처리 과정: 입력: "강아지 키우는 법 알려줘" 1단계: 토큰화 (문장 쪼개기) → ["강아지", "키우는", "법", "알려줘"] 2단계: 숫자로 변환 → "강아지" = [0.2, 0.8, 0.3, ...] "키우는" = [0.5, 0.3, 0.7, ...] "법" = [0.1, 0.2, 0.9, ...] "알려줘" = [0.8, 0.1, 0.4, ...] 3단계: 신경망에서 처리 → 수백만 개의 계산 수행 → 답변 생성용 벡터 생성 4단계: 다시 문자로 변환 → [0.7, 0.3, ...] = "강아지를" [0.2, 0.9, ...] = "키우려면" [0.4, 0.5, ...] = "먼저" ... 최종 출력: "강아지를 키우려면 먼저 ..."

보이지? 중간 과정은 전부 숫자야.

우리가 보는 문자는 입력과 출력뿐이야.

Part 5. 왜 이렇게 복잡하게 만들어?

문자 그대로 규칙 만들어서 처리하면 안 돼?

옛날에는 그렇게 했어. 근데 한계가 명확했어.

규칙 기반 방식의 한계: 규칙 1: "안녕하세요" → "안녕하세요!" 응답 규칙 2: "뭐해" → "저는 AI입니다" 응답 규칙 3: "날씨" → "날씨 정보를 모릅니다" 응답 ... 문제: - 모든 문장에 규칙을 만들 수 없음 - "안녕"과 "안녕하세요"를 다르게 처리 - "뭐하니", "뭐하세요", "뭐하고 있어" 전부 다른 규칙 필요 - 신조어, 은어, 오타 처리 불가 → 규칙이 수백만 개 필요하고, 그래도 부족함

하지만 숫자로 바꿔서 학습하면?

학습 기반 방식의 장점: "안녕" → [0.7, 0.2, ...] "안녕하세요" → [0.71, 0.19, ...] "안뇽" → [0.69, 0.21, ...] → 벡터가 비슷함 = 의미가 비슷함을 자동으로 학습 → 규칙 없이도 비슷하게 처리 가능 → 처음 보는 표현도 비슷한 벡터면 유추해서 처리 "강아지 키우는 법" → [0.3, 0.8, ...] "개 키우기" → [0.31, 0.79, ...] → 같은 의미로 이해함

이게 바로 현대 NLP의 핵심이야.

규칙이 아니라 패턴을 학습하는 거지.

핵심 정리

  • 컴퓨터는 숫자만 이해한다 - 문자는 반드시 숫자로 변환해야 함
  • 단순 번호가 아닌 벡터 - 의미를 담은 여러 숫자(벡터)로 표현
  • AI가 자동으로 학습 - 수백만 문장을 읽으면서 의미 있는 벡터 생성
  • 유사도 계산 가능 - 벡터로 만들면 단어 간 유사성을 수치로 측정 가능
  • 중간 과정은 전부 숫자 - 입력(문자) → 처리(숫자) → 출력(문자)
