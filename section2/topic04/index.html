TOPIC 04

행렬은 왜 쓰는 거야?

Q. 표 형태로 쓰면 뭐가 좋은데?

Part 1. 행렬이 뭔데?

행렬이 뭐야? 표 아니야?

맞아. 행렬은 숫자를 표 형태로 배열한 거야.

행렬 예시: A = | 1 2 3 | | 4 5 6 | → 2행 3열 행렬 (2×3) → 가로줄 = 행(row) → 세로줄 = 열(column) 벡터 vs 행렬: 벡터 = [1, 2, 3] ← 1차원 배열 행렬 = | 1 2 3 | ← 2차원 배열 | 4 5 6 |
벡터로 충분한데 왜 행렬이 필요해?

여러 벡터를 한 번에 처리하려고.

Part 2. AI에서 왜 행렬을 쓰는데?

구체적으로 어디에 쓰이는 거야?

AI에서 행렬은 모든 곳에 쓰여.

1. 데이터 저장 학생 3명의 시험 점수 (국어, 영어, 수학) 학생1: [80, 90, 85] 학생2: [75, 88, 92] 학생3: [90, 85, 88] 행렬로 표현: 국어 영어 수학 학생1 | 80 90 85 | 학생2 | 75 88 92 | 학생3 | 90 85 88 | → 데이터를 구조화해서 한 번에 관리 2. 이미지 데이터 흑백 이미지 4×4 픽셀: | 0 50 100 255 | | 30 80 120 200 | | 60 110 150 180 | | 90 140 180 220 | → 각 숫자가 픽셀의 밝기 → 이미지 전체가 하나의 행렬 3. 가중치(Weight) 뉴럴 네트워크의 연결 강도: 입력층(3개) → 은닉층(2개) W = | 0.5 0.3 0.8 | | 0.2 0.7 0.4 | → 각 값이 연결의 강도를 나타냄 → AI 학습 = 이 행렬 값을 조정하는 것

보이지?

행렬은 AI의 핵심 데이터 구조야.

Part 3. 행렬이 뭐가 좋은데?

벡터 여러 개로 나눠서 저장하면 안 돼?

가능은 한데 엄청 비효율적이야.

벡터 여러 개 행렬 하나
v1 = [80, 90, 85]
v2 = [75, 88, 92]
v3 = [90, 85, 88]
→ 따로따로 관리 		M = | 80 90 85 |
    | 75 88 92 |
    | 90 85 88 |
→ 한 번에 관리
for v in [v1, v2, v3]:
  process(v)
→ 루프 필요 		process(M)
→ 한 번에 처리
느림 빠름 (GPU 가속)

핵심 장점:

1. 병렬 처리 가능 - GPU로 한 번에 계산

2. 코드가 간결 - 루프 없이 한 줄로

3. 수학적으로 명확 - 이론과 구현이 일치

Part 4. 행렬로 뭘 할 수 있어?

행렬로 어떤 계산을 하는데?

기본 연산들을 볼게:

1. 행렬 덧셈 A = | 1 2 | B = | 5 6 | | 3 4 | | 7 8 | A + B = | 1+5 2+6 | = | 6 8 | | 3+7 4+8 | | 10 12 | → 같은 위치끼리 더함 2. 스칼라 곱 A = | 1 2 | | 3 4 | 2 × A = | 2 4 | | 6 8 | → 모든 원소에 2를 곱함 3. 전치(Transpose) A = | 1 2 3 | | 4 5 6 | A^T = | 1 4 | | 2 5 | | 3 6 | → 행과 열을 바꿈 → (i,j) 위치가 (j,i)로 이동

이것들은 기본이고, 가장 중요한 건 행렬 곱셈이야.

그건 다음 토픽에서 자세히 볼게.

Part 5. 실제 AI 코드에서는?

실제로는 어떻게 쓰는 거야?

뉴럴 네트워크 예시를 볼게:

뉴럴 네트워크 한 층: 입력: 100개의 데이터, 각 데이터는 784차원 (28×28 이미지) X = 100×784 행렬 (데이터 100개를 행렬로 표현) W = 784×128 행렬 (가중치) 출력 = X × W = 100×128 행렬 → 한 줄의 코드로 100개 데이터를 동시에 처리! 코드: import numpy as np X = np.random.rand(100, 784) # 입력 데이터 W = np.random.rand(784, 128) # 가중치 output = X @ W # 행렬 곱셈 (@ 연산자) # 결과: (100, 128) 행렬 → GPU가 있으면 엄청 빠름
행렬 없이 하면 어떻게 돼?

루프를 100번 돌려야 해.

행렬 없이 (비효율적): results = [] for i in range(100): result = [] for j in range(128): sum = 0 for k in range(784): sum += X[i][k] * W[k][j] result.append(sum) results.append(result) → 3중 루프 → 엄청 느림 → GPU 못 씀 행렬 사용 (효율적): output = X @ W → 한 줄 → GPU로 병렬 처리 → 수백 배 빠름

Part 6. 행렬 크기 맞추기

행렬 연산할 때 크기가 안 맞으면 어떻게 돼?

에러가 나.

행렬 연산에는 크기 규칙이 있어.

덧셈/뺄셈: 크기가 똑같아야 함 | 1 2 | + | 5 6 | = OK ✓ | 3 4 | | 7 8 | | 1 2 | + | 5 6 7 | = 에러 ✗ | 3 4 | | 8 9 10| 곱셈: A의 열 개수 = B의 행 개수 (2×3) × (3×4) = (2×4) ✓ (2×3) × (4×5) = 에러 ✗ → 중간 숫자(3)가 맞아야 함

AI 코딩할 때 가장 자주 나는 에러가 행렬 크기 불일치야.

항상 크기를 확인하는 습관을 들여야 해.

파이썬에서 크기 확인: import numpy as np A = np.random.rand(100, 784) print(A.shape) # (100, 784) B = np.random.rand(784, 128) print(B.shape) # (784, 128) C = A @ B print(C.shape) # (100, 128) # 크기 안 맞으면 D = np.random.rand(500, 128) E = A @ D # 에러! 784 ≠ 500

핵심 정리

  • 행렬은 2차원 숫자 배열 - 표 형태로 데이터 저장
  • 여러 벡터를 한 번에 처리 - 효율적 데이터 관리
  • AI의 핵심 데이터 구조 - 이미지, 가중치, 배치 데이터 전부
  • 병렬 처리 가능 - GPU로 빠르게 계산
  • 행렬 곱셈이 가장 중요 - 뉴럴 네트워크의 핵심 연산
  • 크기 규칙 중요 - 연산마다 맞춰야 할 크기가 다름
