TOPIC 06

차원이 뭐야?

Q. 3차원까지는 아는데 784차원은 뭐야?

Part 1. 차원이 뭔데?

1차원, 2차원, 3차원은 알겠는데 AI에서는 784차원이라는데?

차원 = 정보를 나타내는 숫자의 개수

물리적 공간하고는 좀 달라.

물리적 차원: 1차원 = 선 (x) 2차원 = 평면 (x, y) 3차원 = 공간 (x, y, z) → 우리가 볼 수 있는 건 여기까지 수학적 차원: 4차원 = [x, y, z, w] 5차원 = [a, b, c, d, e] 784차원 = [x1, x2, x3, ..., x784] → 그냥 숫자가 784개 있다는 뜻 → 시각화는 못해도 계산은 가능!

이해됐어?

AI에서 차원 = 벡터에 들어있는 숫자 개수

Part 2. 왜 차원이 이렇게 높아?

784차원은 왜 필요한 거야?

MNIST 손글씨 숫자 데이터셋 얘기야.

28×28 픽셀 이미지를 벡터로 펼친 거야.

MNIST 이미지: 28×28 = 784 픽셀 원본 (행렬): | p1 p2 p3 ... p28 | | p29 p30 p31 ... p56 | | ... | | p757 ... p784 | 벡터로 펼침: [p1, p2, p3, ..., p784] → 784차원 벡터! → 각 차원이 하나의 픽셀 값 컬러 이미지면? 28×28×3 (RGB) = 2352차원 → 더 많은 정보 = 더 높은 차원

원칙:

표현하려는 정보가 많을수록 차원이 높아져.

Part 3. 차원이 높으면 뭐가 문제야?

차원이 높으면 더 좋은 거 아니야? 정보가 많으니까

아니야. "차원의 저주"라는 게 있어.

차원의 저주 (Curse of Dimensionality): 1차원: 선 위에 점 10개 → 빽빽함 2차원: 평면에 점 10개 → 듬성듬성 3차원: 공간에 점 10개 → 거의 빈 공간 1000차원: 점 10개 → 거의 모두가 외딴 섬 문제: 1. 데이터가 너무 희소해짐 2. 거리 개념이 무의미해짐 3. 필요한 데이터가 기하급수적으로 증가 예시: - 1차원에서 10개 구간 채우기 → 10개 필요 - 2차원에서 10×10 채우기 → 100개 필요 - 3차원에서 10×10×10 채우기 → 1000개 필요 - 784차원? → 10^784개 필요 (불가능!)

그래서 차원 축소 기법이 중요해.

Part 4. 차원을 줄일 수 있어?

차원을 줄이면 정보가 사라지잖아?

중요한 정보만 남기고 줄이는 거야.

예시: 학생 평가 원본 데이터 (5차원): [국어, 영어, 수학, 과학, 사회] 발견: 과목들이 상관관계가 높음 → 국어 잘하면 영어도 잘함 → 수학 잘하면 과학도 잘함 차원 축소 (2차원): ["언어 능력", "이과 능력"] → 5차원 → 2차원 → 대부분의 정보는 유지하면서 차원 감소! 방법: - PCA (주성분 분석) - t-SNE - UMAP - 오토인코더

실제 사용:

1. 시각화 - 고차원 데이터를 2D/3D로 줄여서 그림

2. 전처리 - 학습 전에 불필요한 차원 제거

3. 압축 - 저장 공간 절약

Part 5. 워드 임베딩은 왜 300차원?

Word2Vec은 보통 300차원이라던데 왜 그 숫자야?

경험적으로 찾은 적당한 크기야.

차원 장점 단점
50차원 빠름, 가벼움 표현력 부족
300차원 표현력 좋음, 속도 적당 균형있음
1000차원 최고 표현력 느림, 무거움
최신 모델들: Word2Vec: 300차원 GloVe: 50~300차원 BERT: 768차원 GPT-2: 768~1536차원 GPT-3: 12288차원 (큰 버전) → 모델이 커질수록 차원도 증가 → 더 복잡한 의미 표현 가능

차원이 높을수록 성능은 좋지만 계산량도 증가해.

적절한 균형을 찾는 게 중요해.

Part 6. 차원 다루기

파이썬에서 차원은 어떻게 확인해?

NumPy로 쉽게 확인할 수 있어:

import numpy as np # 1차원 벡터 v = np.array([1, 2, 3]) print(v.shape) # (3,) → 3차원 벡터 # 2차원 행렬 m = np.array([[1, 2, 3], [4, 5, 6]]) print(m.shape) # (2, 3) → 2행 3열 # 고차원 벡터 img = np.random.rand(784) print(img.shape) # (784,) → 784차원 벡터 # 이미지를 벡터로 펼치기 img_2d = np.random.rand(28, 28) # 28×28 이미지 img_flat = img_2d.flatten() # 벡터로 펼침 print(img_flat.shape) # (784,) # 차원 축소 (간단한 예) from sklearn.decomposition import PCA data = np.random.rand(100, 784) # 100개, 784차원 pca = PCA(n_components=50) # 50차원으로 축소 reduced = pca.fit_transform(data) print(reduced.shape) # (100, 50)

핵심 정리

  • 차원 = 숫자 개수 - 벡터에 들어있는 값의 개수
  • 고차원은 자연스러움 - 이미지, 텍스트는 본질적으로 고차원
  • 차원의 저주 - 차원이 높으면 데이터가 희소해짐
  • 차원 축소 필요 - PCA 등으로 중요한 정보만 추출
  • 적절한 균형 - 표현력 vs 계산량의 트레이드오프
  • 300차원이 표준 - 많은 임베딩 모델의 기본값
