TOPIC 13

소프트맥스가 뭐야?

Q. 숫자를 확률로 어떻게 바꿔?

Part 1. 소프트맥스가 뭔데?

Softmax가 뭐야?

숫자를 확률로 바꾸는 함수

분류 문제의 마지막 단계야.

문제: 뉴럴 네트워크 출력 (로짓): [2.0, 1.0, 0.1] → 이건 확률이 아니야 → 음수도 가능, 합이 1도 아님 Softmax 적용: Softmax(x_i) = e^(x_i) / Σe^(x_j) 계산: e^2.0 = 7.39 e^1.0 = 2.72 e^0.1 = 1.11 합 = 7.39 + 2.72 + 1.11 = 11.22 확률: 7.39/11.22 = 0.66 (66%) 2.72/11.22 = 0.24 (24%) 1.11/11.22 = 0.10 (10%) 결과: [0.66, 0.24, 0.10] → 합 = 1 ✓ → 모두 양수 ✓ → 확률 분포!

Part 2. 왜 e를 쓰나?

왜 지수 함수를 쓰는 거야? 그냥 나누면 안 돼?

여러 이유가 있어:

이유 1: 음수 처리 입력: [-2, 1, 3] 그냥 나누기: -2 + 1 + 3 = 2 -2/2 = -1 ← 음수! 확률 불가 e 사용: e^-2 = 0.14 e^1 = 2.72 e^3 = 20.09 → 모두 양수 이유 2: 큰 차이 강조 입력: [3, 2, 1] 그냥 비율: 3:2:1 Softmax: 0.67:0.24:0.09 → 가장 큰 값이 더 두드러짐 이유 3: 미분이 깔끔 Softmax 미분 = Softmax × (1 - Softmax) → 역전파가 쉬워짐

Part 3. Temperature 파라미터

Temperature는 뭐야?

확률 분포의 "뾰족함"을 조절

Temperature Softmax: Softmax(x_i, T) = e^(x_i/T) / Σe^(x_j/T) 로짓: [3, 2, 1] T=1 (기본): [0.67, 0.24, 0.09] T=0.5 (낮음 - 더 확신): [0.84, 0.14, 0.02] → 가장 큰 값에 집중 T=2 (높음 - 덜 확신): [0.46, 0.32, 0.22] → 고르게 분산 T→0 (극단): [1.0, 0.0, 0.0] → argmax와 같음 T→∞ (극단): [0.33, 0.33, 0.33] → 균등 분포

Part 4. 실제 사용

어디에 쓰이는 거야?

거의 모든 분류 문제:

1. 이미지 분류 입력: 고양이 사진 로짓: [5.2, 2.1, 0.8] (고양이, 개, 토끼) Softmax: [0.91, 0.08, 0.01] → "91% 확신해서 고양이" 2. 언어 모델 다음 단어 예측: "I love ___" 로짓: {apple: 2.1, music: 1.5, ...} Softmax: {apple: 0.3, music: 0.2, ...} → 확률에 따라 단어 샘플링 3. 멀티클래스 분류 10개 숫자 분류 (MNIST): 로짓: [1.2, 0.5, 3.5, ..., 0.1] Softmax: [0.05, 0.02, 0.45, ..., 0.01] → 가장 높은 클래스 선택

Part 5. 코드 구현

어떻게 쓰는 거야?

PyTorch 예시:

import torch import torch.nn as nn import torch.nn.functional as F # 방법 1: 함수 사용 logits = torch.tensor([2.0, 1.0, 0.1]) probs = F.softmax(logits, dim=0) print(probs) # tensor([0.6590, 0.2424, 0.0986]) # 방법 2: 직접 구현 def softmax(x): exp_x = torch.exp(x) return exp_x / exp_x.sum() # Temperature 조절 def softmax_temp(x, T=1.0): exp_x = torch.exp(x / T) return exp_x / exp_x.sum() # T=0.5 (더 확신) print(softmax_temp(logits, T=0.5)) # tensor([0.7763, 0.1865, 0.0372]) # T=2.0 (덜 확신) print(softmax_temp(logits, T=2.0)) # tensor([0.5118, 0.3033, 0.1849]) # 배치 처리 logits_batch = torch.randn(32, 10) # 배치 32, 클래스 10 probs_batch = F.softmax(logits_batch, dim=1) # (32, 10) - 각 행의 합 = 1

Part 6. LogSoftmax

LogSoftmax는 뭐야?

log(Softmax(x))

수치 안정성을 위해 써.

문제: 큰 숫자: e^1000 = overflow! 해결: LogSoftmax LogSoftmax(x_i) = x_i - log(Σe^(x_j)) → e를 직접 계산 안 함 → 수치적으로 안정 사용: # Cross-Entropy Loss criterion = nn.CrossEntropyLoss() # 내부적으로 LogSoftmax + NLLLoss 사용 # 더 안정적!

핵심 정리

  • Softmax = 확률 변환 - 로짓을 확률로
  • 지수 함수 사용 - e^x로 양수 보장
  • 합 = 1 - 확률 분포의 기본 조건
  • Temperature로 조절 - 확신의 정도 변경
  • 분류 문제 필수 - 마지막 층에 사용
  • LogSoftmax도 있음 - 수치 안정성
